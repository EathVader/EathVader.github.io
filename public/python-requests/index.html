Python Requests 深度解析

来源：Anthony Shaw · 2026-04-02

Requests 是 Python 最流行的 HTTP 库。很多人用它发几个请求就满足了，但它的能力远不止于此。本文将深入解析 requests 的核心机制。

发起请求的多种方式

Requests 提供了便捷的顶层函数：

Python
import requests

# 便捷方法
r = requests.get('https://api.example.com/data')
r = requests.post('https://api.example.com/data', json={'key': 'value'})
r = requests.put('https://api.example.com/data/1', json={'key': 'new'})
r = requests.delete('https://api.example.com/data/1')

这些都是 requests.request() 的友好封装，HTTP 动词作为第一个参数传入。不要写冗长的 if/elif 判断 HTTP 动词——request() 本身就是主 API。

关键发现：调用 requests.get() 等方法时，底层会创建一个 Session 对象。如果你要对同一个端点发起多次请求，直接使用 Session 更好——它能保持 TCP 连接（keep-alive）、维护 cookie jar、记住每次请求的偏好设置。

深入理解 Session 对象

常见反模式

Python — 不好的写法
# ❌ 每次请求都创建新的 TCP 连接
r = requests.post('https://api.example.com/data',
                  json={'key': 'value'},
                  headers={'Authorization': 'Bearer xxx'})
data = json.loads(r.text)

正确的 Session 用法

Python — 好的写法
# ✅ 使用 Session 保持连接、管理 Cookie
with requests.Session() as sess:
    # 设置默认 headers（整个 Session 生效）
    sess.headers.update({
        'Authorization': 'Bearer xxx',
        'Content-Type': 'application/json'
    })
    
    # 多次请求复用同一个 TCP 连接
    r = sess.post('https://api.example.com/data',
                  json={'key': 'value'})
    data = r.json()  # 内置 JSON 解析
    
    r = sess.get('https://api.example.com/data/1')
    item = r.json()

Session 的默认 Headers

Session 初始化时会设置一组默认 headers：

Python
sess = requests.Session()
print(sess.headers)
# {
#     'Accept': '*/*',
#     'Accept-Encoding': 'gzip, deflate',  # 自动支持 gzip 响应
#     'Connection': 'keep-alive',           # 默认保持连接
#     'User-Agent': 'python-requests/x.x.x'
# }
⚠️ 注意：避免直接给 sess.headers 赋新字典（如 sess.headers = {...}），因为默认属性是一个大小写不敏感的字典（CaseInsensitiveDict），直接赋值会丢失这个特性。应该使用 sess.headers.update()。

深入理解 PreparedRequest

当你继续剥开洋葱，Session.request() 会创建一个 PreparedRequest 对象。你可以手动构造这些对象，特别是在需要发送特定二进制 payload 和 headers 时。

Python
from requests import Request, Session

sess = Session()

# 创建 Request 对象
req = Request('POST',
              'https://api.example.com/upload',
              headers={'Content-Type': 'application/octet-stream'},
              data=b'binary payload here')

# 准备请求（PreparedRequest）
prepared = req.prepare()

# 发送准备好的请求
response = sess.send(prepared)

这种方式在你需要对请求进行精细控制时非常有用，比如修改已准备好的请求：

Python
prepared = req.prepare()
# 可以在发送前修改
prepared.headers['X-Custom-Header'] = 'value'
prepared.body = modify_payload(prepared.body)

response = sess.send(prepared)

处理大型请求：流和迭代器

流式上传

Requests 会自动检测 data 参数是否是迭代器（如文件流对象、BytesIO 或 StringIO），并使用流式上传：

Python
# 流式上传大文件
with open('large_file.bin', 'rb') as f:
    requests.post('https://api.example.com/upload', data=f)

# Requests 自动检测到文件对象，使用 Transfer-Encoding: chunked

流式下载

使用 stream=True 参数可以让 iter_content 从打开的连接中迭代数据，而不是一次性加载到内存：

Python — 流式下载大文件
# 流式下载
r = requests.get('https://example.com/large_file.zip', stream=True)

with open('downloaded.zip', 'wb') as f:
    for chunk in r.iter_content(chunk_size=8192):
        if chunk:
            f.write(chunk)
关键区别：不使用 stream=True 时，iter_content 仍然可以工作，但会先把整个响应下载到内存中（执行时间可能翻倍）。下载大文件时一定要用 stream=True。

流式下载 JSON 流

Python — 处理 JSON Lines
r = requests.get('https://api.example.com/stream', stream=True)

for line in r.iter_lines():
    if line:
        data = json.loads(line)
        process(data)

最佳实践总结

场景推荐做法
单个请求requests.get() 等便捷方法
同一端点多次请求使用 Session，复用 TCP 连接
需要认证在 Session 上设置默认 headers 或使用 auth 参数
上传大文件传入文件对象作为 data，自动流式上传
下载大文件stream=True + iter_content()
精细控制请求手动创建 PreparedRequest
JSON 请求/响应使用 json= 参数和 r.json()

完整示例：健壮的 API 客户端

Python
import requests
from requests.adapters import HTTPAdapter
from urllib3.util.retry import Retry

class APIClient:
    def __init__(self, base_url, api_key):
        self.base_url = base_url
        self.session = requests.Session()
        
        # 设置默认 headers
        self.session.headers.update({
            'Authorization': f'Bearer {api_key}',
            'Accept': 'application/json',
        })
        
        # 配置重试策略
        retry = Retry(
            total=3,
            backoff_factor=1,
            status_forcelist=[429, 500, 502, 503, 504],
        )
        adapter = HTTPAdapter(max_retries=retry)
        self.session.mount('https://', adapter)
        self.session.mount('http://', adapter)
    
    def get(self, path, **kwargs):
        r = self.session.get(
            f'{self.base_url}/{path}',
            timeout=10,
            **kwargs
        )
        r.raise_for_status()
        return r.json()
    
    def post(self, path, data=None, json=None):
        r = self.session.post(
            f'{self.base_url}/{path}',
            json=json,
            data=data,
            timeout=10,
        )
        r.raise_for_status()
        return r.json()
    
    def download(self, path, dest_path):
        """流式下载文件"""
        r = self.session.get(
            f'{self.base_url}/{path}',
            stream=True,
            timeout=30,
        )
        r.raise_for_status()
        with open(dest_path, 'wb') as f:
            for chunk in r.iter_content(chunk_size=8192):
                f.write(chunk)
    
    def close(self):
        self.session.close()
    
    def __enter__(self):
        return self
    
    def __exit__(self, *args):
        self.close()

总结

核心要点

  1. Session 是核心：多次请求用 Session，复用 TCP 连接、管理 Cookie、保持默认设置
  2. 不要直接赋值 headers：用 sess.headers.update() 而不是 sess.headers = {...}
  3. PreparedRequest：需要精细控制时手动构造
  4. 流式上传：传入文件对象自动流式上传
  5. 流式下载：stream=True + iter_content() 避免内存爆炸
  6. JSON 原生支持：json= 参数自动编码，r.json() 自动解析
