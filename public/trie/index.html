1 什么是 Trie 树 / What is Trie

📖

类比 / Analogy

Trie 树就像一本按字母排列的字典：所有以 "a" 开头的词在同一分支，以 "ab" 开头的在更深的分支。查找 "apple" 时，你只需沿着 a→p→p→l→e 走下去，不需要和其他单词比较。

📐 Trie 树的三条性质
  1. 根节点不包含字符，其余每个节点包含一个字符
  2. 从根到某节点的路径上的字符连接起来 = 该节点对应的字符串
  3. 每个节点的所有子节点包含的字符互不相同
公共前缀共享节点：tea/ten 共享 t→e，in/inn 共享 i→n
⚡ 时间复杂度

查找/插入/删除：O(L)，L 为字符串长度。与字典中有多少个词无关！比哈希表更适合前缀搜索。

5 代码实现 / Implementation

class TrieNode {
    TrieNode[] children = new TrieNode[26]; // a-z
    boolean isEnd = false;
    int count = 0; // 经过此节点的单词数
}

class Trie {
    TrieNode root = new TrieNode();

    // 插入单词
    void insert(String word) {
        TrieNode node = root;
        for (char c : word.toCharArray()) {
            int i = c - 'a';
            if (node.children[i] == null)
                node.children[i] = new TrieNode();
            node = node.children[i];
            node.count++;
        }
        node.isEnd = true;
    }

    // 精确查找
    boolean search(String word) {
        TrieNode node = root;
        for (char c : word.toCharArray()) {
            int i = c - 'a';
            if (node.children[i] == null) return false;
            node = node.children[i];
        }
        return node.isEnd;
    }

    // 前缀计数（有多少单词以 prefix 开头）
    int countPrefix(String prefix) {
        TrieNode node = root;
        for (char c : prefix.toCharArray()) {
            int i = c - 'a';
            if (node.children[i] == null) return 0;
            node = node.children[i];
        }
        return node.count;
    }
}
💾 空间优化

固定 26 个子节点数组简单但浪费空间（大量 null）。实际中常用 HashMap<Character, TrieNode> 代替数组，或使用双数组 Trie（见第 7 节）。

6 Trie vs HashMap

初学者常问：为什么不直接用 HashMap 存所有单词？

Trie 树HashMap
精确查找O(L)O(L)（计算哈希）
前缀搜索✅ O(L + k)❌ 需遍历所有键 O(N)
自动补全✅ 天然支持❌ 不支持
按字典序遍历✅ DFS 即可❌ 需排序
空间（大量公共前缀）✅ 共享节点每个词独立存储
空间（无公共前缀）❌ 每字符一节点✅ 更紧凑
🎯 选择原则

用 Trie：需要前缀搜索、自动补全、按字典序遍历
用 HashMap：只需精确查找，词之间没有公共前缀

7 应用：敏感词过滤 / Sensitive Word Filter

🛡️

原理 / How it works

把所有敏感词插入 Trie 树。对于待检测文本，从每个位置开始，沿 Trie 树走，看能否走到某个单词结尾。如果能，说明包含敏感词。

从每个位置开始在 Trie 中走，走到单词结尾即命中敏感词
// 敏感词过滤（简化版）
List<String> filter(String text, Trie trie) {
    List<String> hits = new ArrayList<>();
    for (int i = 0; i < text.length(); i++) {
        TrieNode node = trie.root;
        for (int j = i; j < text.length(); j++) {
            int idx = text.charAt(j) - 'a';
            if (node.children[idx] == null) break;
            node = node.children[idx];
            if (node.isEnd) hits.add(text.substring(i, j + 1));
        }
    }
    return hits;
}
🔧 去噪处理

用户可能用 "芒**果"、"芒 果" 绕过过滤。解决方案：过滤前先去除空格和特殊符号：text = text.replaceAll("[\\s&%$@*]+", "")

8 应用：IP 地址查询 / IP Lookup

给定一个 IP 地址，查询它属于哪个 ISP（中国移动/电信/联通）。

💡 核心思路

将 IP 地址转为 32 位二进制，构建一棵二叉 Trie 树（每个节点只有 0/1 两个子节点）。每个 ISP 的 IP 段（CIDR 格式）对应树中的一条路径。查询时沿二进制位走到叶节点即可。

优势：查询时间 O(32)（固定 32 步），与 ISP 数量无关，比遍历所有 IP 段快得多。

9 双数组 Trie / Double-Array Trie

普通 Trie 树的问题：每个节点存一个 26 元素数组，空间浪费严重。双数组 Trie（DAT）用两个整数数组 base[] 和 check[] 压缩存储整棵树。

🔑 核心转移方程
base[r] + c = s      // 从状态 r 经字符 c 转移到状态 s
check[s] = r         // 验证转移有效性

其中 c 为字符的整数编码。两个数组就能表示整棵树的所有转移关系。

实现方式查询时间空间典型应用
数组 TrieO(1)/字符大（大量 null）简单场景
HashMap TrieO(1)/字符中mmseg4j
链表 TrieO(n)/字符小内存受限
双数组 TrieO(1)/字符最小Ansj 分词、搜索引擎

10 总结 / Summary

操作时间复杂度说明
插入O(L)L = 字符串长度
查找O(L)与字典大小无关
前缀搜索O(L + k)k = 结果数量
空间O(N × Σ)N = 节点数，Σ = 字符集大小
🌍 Trie 树的实际应用
  • 搜索引擎：输入框自动补全（Google、百度）
  • 内容安全：敏感词/违禁词过滤
  • 网络路由：IP 路由表最长前缀匹配
  • 中文分词：Ansj、jieba 等分词工具的词典
  • 拼写检查：Word、IDE 的拼写纠错
