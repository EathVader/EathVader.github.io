1 为什么需要线段树 / Why Segment Tree

🏢

类比 / Analogy

想象你是一家连锁店的老板，有 10 万家门店。每天你需要查询"第 3000~8000 家店的总销售额"，还要随时修改某家店的数据。暴力遍历太慢，前缀和不支持修改——线段树就是为此而生。

给定数组 a[0..n-1]，我们需要高效支持两种操作：

操作暴力前缀和线段树
区间求和 query(l, r)O(n)O(1)O(log n)
单点修改 update(i, v)O(1)O(n) 重建O(log n)
区间修改 + 区间查询O(n)O(n)O(log n)
🎯 线段树的定位

当你需要同时支持修改和区间查询时，线段树是最通用的选择。它在两种操作之间取得了完美平衡。

2 什么是线段树 / What is Segment Tree

📐 线段树的三条性质
  1. 线段树是一棵完全二叉树（用数组存储，节点 i 的子节点为 2i 和 2i+1）
  2. 每个节点代表一个区间，存储该区间的聚合值（如求和、最大值）
  3. 叶节点对应原数组的单个元素，内部节点 = 左子 + 右子
💾 存储方式

用数组 tree[1..4n] 存储。节点编号 i 的左子为 2i，右子为 2i+1。无需指针，缓存友好。开 4n 大小的数组即可保证不越界。

// 建树：O(n)
void build(int node, int l, int r) {
    if (l == r) {
        tree[node] = a[l];  // 叶节点
        return;
    }
    int mid = (l + r) / 2;
    build(2 * node, l, mid);       // 建左子树
    build(2 * node + 1, mid+1, r); // 建右子树
    tree[node] = tree[2*node] + tree[2*node+1]; // 合并
}
⏱️ 时间复杂度

建树 O(n)：每个元素恰好被访问一次作为叶节点，每个内部节点做一次加法。总共 2n-1 个节点。

5 区间查询 / Range Query

查询 query(1, 4)（求 a[1]+a[2]+a[3]+a[4]）的过程：

结果 = 5 + 1 + 13 = 19　　三种情况：完全包含→直接返回 | 完全不交→返回0 | 部分相交→递归两子
// 区间查询：O(log n)
int query(int node, int l, int r, int ql, int qr) {
    if (ql <= l && r <= qr) return tree[node]; // 完全包含
    if (ql > r || qr < l) return 0;            // 完全不交
    int mid = (l + r) / 2;                     // 部分相交：递归两子
    return query(2*node, l, mid, ql, qr)
         + query(2*node+1, mid+1, r, ql, qr);
}
💡 为什么是 O(log n)？

每层最多访问 4 个节点（左右边界各最多拆分一次），树高 O(log n)，所以总复杂度 O(log n)。这个常数 4 的证明可以用归纳法。

// 单点更新：O(log n)
void update(int node, int l, int r, int idx, int val) {
    if (l == r) { tree[node] = val; return; } // 到达叶节点
    int mid = (l + r) / 2;
    if (idx <= mid) update(2*node, l, mid, idx, val);
    else update(2*node+1, mid+1, r, idx, val);
    tree[node] = tree[2*node] + tree[2*node+1]; // 回溯合并
}

7 懒传播 / Lazy Propagation

如果要对整个区间加一个值（如 a[1..4] 都加 3），逐个更新是 O(n log n)。懒传播让区间修改也是 O(log n)。

📝

类比 / Analogy

老板说"3000~8000 号门店全部涨价 10%"。你不会跑去通知每家店，而是在区域经理那里贴个便签："你管的店都涨价 10%"。只有当需要查询某家具体门店时，才把便签传递下去。这就是懒传播。

// 懒传播核心：下推标记
void pushDown(int node, int l, int r) {
    if (lazy[node] != 0) {
        int mid = (l + r) / 2;
        // 传给左子
        tree[2*node] += lazy[node] * (mid - l + 1);
        lazy[2*node] += lazy[node];
        // 传给右子
        tree[2*node+1] += lazy[node] * (r - mid);
        lazy[2*node+1] += lazy[node];
        // 清除自己
        lazy[node] = 0;
    }
}

// 区间修改：[ql, qr] 每个元素 +val
void rangeUpdate(int node, int l, int r, int ql, int qr, int val) {
    if (ql <= l && r <= qr) {
        tree[node] += val * (r - l + 1);
        lazy[node] += val;
        return;
    }
    pushDown(node, l, r);
    int mid = (l + r) / 2;
    if (ql <= mid) rangeUpdate(2*node, l, mid, ql, qr, val);
    if (qr > mid)  rangeUpdate(2*node+1, mid+1, r, ql, qr, val);
    tree[node] = tree[2*node] + tree[2*node+1];
}
🎯 懒传播的本质

延迟计算：修改时只更新必要的节点并打标记，查询时才把标记下推。这样区间修改和区间查询都是 O(log n)。

8 代码实现 / Implementation

完整的线段树模板（支持区间修改 + 区间查询）：

class SegmentTree {
    int[] tree, lazy;
    int n;

    SegmentTree(int[] a) {
        n = a.length;
        tree = new int[4 * n];
        lazy = new int[4 * n];
        build(1, 0, n - 1, a);
    }

    void build(int node, int l, int r, int[] a) {
        if (l == r) { tree[node] = a[l]; return; }
        int mid = (l + r) / 2;
        build(2 * node, l, mid, a);
        build(2 * node + 1, mid + 1, r, a);
        tree[node] = tree[2 * node] + tree[2 * node + 1];
    }

    void pushDown(int node, int l, int r) {
        if (lazy[node] != 0) {
            int mid = (l + r) / 2;
            apply(2 * node, l, mid, lazy[node]);
            apply(2 * node + 1, mid + 1, r, lazy[node]);
            lazy[node] = 0;
        }
    }

    void apply(int node, int l, int r, int val) {
        tree[node] += val * (r - l + 1);
        lazy[node] += val;
    }

    void update(int node, int l, int r, int ql, int qr, int val) {
        if (ql <= l && r <= qr) { apply(node, l, r, val); return; }
        pushDown(node, l, r);
        int mid = (l + r) / 2;
        if (ql <= mid) update(2 * node, l, mid, ql, qr, val);
        if (qr > mid)  update(2 * node + 1, mid + 1, r, ql, qr, val);
        tree[node] = tree[2 * node] + tree[2 * node + 1];
    }

    int query(int node, int l, int r, int ql, int qr) {
        if (ql <= l && r <= qr) return tree[node];
        if (ql > r || qr < l) return 0;
        pushDown(node, l, r);
        int mid = (l + r) / 2;
        return query(2 * node, l, mid, ql, qr)
             + query(2 * node + 1, mid + 1, r, ql, qr);
    }

    // Public API
    void rangeAdd(int l, int r, int val) { update(1, 0, n-1, l, r, val); }
    int rangeSum(int l, int r) { return query(1, 0, n-1, l, r); }
}

9 经典应用 / Applications

🔢 区间最值查询 (RMQ)

把合并操作从 + 改为 max 或 min，即可支持区间最大/最小值查询。

// 只需修改合并方式
tree[node] = Math.max(tree[2*node], tree[2*node+1]);

📊 逆序对计数

用线段树维护值域（每个值出现的次数），从右往左扫描数组，对每个元素查询比它小的已出现元素个数。

🎯 动态第 K 小

权值线段树 + 二分：在线段树上二分查找第 K 小的元素。如果左子树的元素个数 ≥ K，递归左子；否则递归右子，K 减去左子树大小。

📐 扫描线算法

求多个矩形的面积并/周长并。用线段树维护 y 轴上被覆盖的长度，从左到右扫描 x 轴事件。

🧩 线段树的灵活性

线段树的核心是分治 + 合并。只要你的问题满足"区间可合并"（如求和、最值、GCD、矩阵乘法），就能用线段树解决。

🎯 选择原则

只需单点修改 + 区间查询：树状数组（代码更短）
需要区间修改：线段树（最通用）
只需区间查询，无修改：前缀和 / 稀疏表
在线竞赛求快：分块（思维简单，常数大）
