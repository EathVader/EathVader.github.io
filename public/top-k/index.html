1 什么是 Top K 问题 / What is Top K

🏅

类比 / Analogy

全校 1000 名学生参加考试，你只需要找出前 10 名。你不需要给所有人排名——只需要维护一个"前 10 名候选名单"，每来一个新成绩，如果比名单中最低分高，就替换掉最低分。

📐 三种经典方法
  1. 全排序：排序后取前 K 个 — O(n log n)，简单但慢
  2. 小顶堆：维护大小为 K 的堆 — O(n log K)，适合流式数据
  3. 快速选择：基于 partition — 平均 O(n)，最快但不稳定

2 小顶堆方法 / Min-Heap Approach

🪣

类比 / Analogy

想象一个只能装 K 个球的桶，桶口朝上，最小的球总在最上面。每来一个新球：如果比桶顶的球大，就把桶顶扔掉，放入新球。最终桶里就是最大的 K 个球。

小顶堆始终保持 K 个最大元素，堆顶是"门槛"
// 小顶堆找 Top K 最大 / Min-heap for Top K largest
int[] topK(int[] nums, int k) {
    PriorityQueue<Integer> heap = new PriorityQueue<>(); // 小顶堆
    for (int num : nums) {
        heap.offer(num);
        if (heap.size() > k) heap.poll(); // 弹出最小的
    }
    return heap.stream().mapToInt(i -> i).toArray();
}
⚡ 为什么用小顶堆而不是大顶堆？

找 Top K 最大用小顶堆：堆顶是 K 个中最小的，新元素只需和堆顶比较。如果用大顶堆，需要把所有 n 个元素都放进去，复杂度变成 O(n log n)。

3 快速选择 / Quick Select

随机选 pivot
→
partition
小的左边，大的右边
→
pivot 位置 = k?
是→返回；否→递归一侧
partition 后 pivot 右边恰好有 K 个元素，即为 Top K
// Quick Select 找第 K 大 / Quick select for Kth largest
int findKthLargest(int[] nums, int k) {
    return quickSelect(nums, 0, nums.length - 1, nums.length - k);
}

int quickSelect(int[] nums, int lo, int hi, int k) {
    int pivot = nums[hi], i = lo;
    for (int j = lo; j < hi; j++) {
        if (nums[j] <= pivot) { swap(nums, i, j); i++; }
    }
    swap(nums, i, hi);
    if (i == k) return nums[i];
    return i < k ? quickSelect(nums, i + 1, hi, k)
                 : quickSelect(nums, lo, i - 1, k);
}
⚠️ 最坏情况

Quick Select 平均 O(n)，但最坏 O(n²)（每次 pivot 选到最大/最小值）。解决方案：随机选 pivot 或使用 Median of Medians 保证 O(n)。

5 应用场景 / Applications

🌍 Top K 的实际应用
  • 搜索引擎：返回相关度最高的 K 个结果
  • 推荐系统：推荐评分最高的 K 个商品
  • 监控系统：找出 CPU 占用最高的 K 个进程
  • 数据分析：找出销量最高的 K 个产品
  • 热搜榜：实时统计搜索频率最高的 K 个词

6 总结 / Summary

方法时间空间适用场景
小顶堆O(n log K)O(K)流式数据、K ≪ n
Quick Select平均 O(n)O(1)内存数组、只需第 K 个
排序O(n log n)O(1)小数据、需完整排序
🎯 面试技巧

面试中遇到 Top K 问题，先说小顶堆方案（最通用），再提Quick Select（展示深度）。如果面试官追问流式数据，堆方案是唯一选择。
