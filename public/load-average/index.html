一分钟理解负载 Load Average

来源：架构师之路 · 作者：58沈剑

什么是 Load Average？

系统负载（System Load）是系统CPU繁忙程度的度量，即有多少进程在等待被CPU调度（进程等待队列的长度）。

平均负载（Load Average）是一段时间内系统的平均负载，这个一段时间一般取 1分钟、5分钟、15分钟。

如何查看 Load？

$ uptime
13:53:39 up 10 days, 2:15, 1 user, load average: 1.5, 2.5, 5.5

如上所示，dev02机器 1分钟平均负载、5分钟平均负载、15分钟平均负载 分别是 1.5、2.5、5.5

top、uptime、w 等命令都可以查看系统负载。

Load 的数值是什么含义？

把 CPU 比喻成一条（单核）马路，进程任务 比喻成马路上跑着的 汽车，Load 则表示 马路的繁忙程度。

Load == 2，双核，没有等待。所以 Load 应该与 CPU 核数比较，而不是与 1 比较。

什么样的 Load 值得警惕？（单核）

Load < 0.7
系统很闲，马路上没什么车
考虑多部署一些服务
0.7 < Load < 1
系统状态不错
马路可以轻松应对
Load == 1
系统马上要处理不过来了
赶紧找一下原因
Load > 5
马路已经非常繁忙了
每辆汽车都无法很快运行

不同 Load 值组合说明什么？

结合 1分钟、5分钟、15分钟 三个值的具体情况来分析：

场景 1 抖动/拥塞前兆
1分钟 > 5 5分钟 < 1 15分钟 < 1

短期内繁忙，中长期空闲

场景 2 拥塞的开始
1分钟 > 5 5分钟 > 1 15分钟 < 1

短期内繁忙，中期内紧张

场景 3 正在拥塞
1分钟 > 5 5分钟 > 5 15分钟 > 5

短中长期都繁忙，系统正在拥塞

场景 4 拥塞正在好转
1分钟 < 1 5分钟 > 1 15分钟 > 5

短期内空闲，中长期繁忙，不用紧张

总结

  • Load Average 是 CPU 繁忙程度的度量，即等待被调度的进程数
  • 三个值分别代表 1分钟、5分钟、15分钟 的平均负载
  • Load 应该与 CPU 核数比较，单核 Load > 1 表示有等待
  • Load < 0.7 很闲，0.7~1 正常，==1 需要关注，>5 非常繁忙
  • 通过三个时间维度的组合，可以判断系统是"抖动"、"拥塞开始"还是"正在好转"

赶快 uptime 一下，看一下自己系统的负载吧！
