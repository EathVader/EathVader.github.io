网络协议模型：七层、五层、四层协议

来源：buknow的博客 · 2018-07-21

一、OSI七层模型

OSI（Open System Interconnect）是开放式系统互联的缩写。OSI七层参考模型的各个层次的划分遵循下列原则：

  • 同一层中的各网络节点都有相同的层次结构，具有同样的功能
  • 同一节点内相邻层之间通过接口进行通信
  • 七层结构中的每一层使用下一层提供的服务，并且向其上层提供服务
  • 不同节点的同等层按照协议实现对等层之间的通信
层次 名称 功能 协议示例 数据单位
7 应用层 为用户提供网络服务 HTTP, FTP, SMTP, DNS 报文
6 表示层 数据格式转换、加密解密 JPEG, ASCII, SSL 报文
5 会话层 会话管理、同步 RPC, SQL 报文
4 传输层 端到端传输、流量控制 TCP, UDP, SPX 段(segment)/数据报(datagram)
3 网络层 路由选择、逻辑寻址 IP, IPX 数据包(packet)
2 数据链路层 帧传输、物理寻址 ATM, FDDI, Ethernet 帧(frame)
1 物理层 比特传输、物理接口 Rj45, 802.3 比特(bit)

二、五层模型

五层模型是OSI和TCP/IP的综合，是业界产生的非官方协议模型。为了方便把下两层称为网络接口层。

五层 对应OSI 对应TCP/IP
应用层 5-7层 应用层
传输层 传输层 传输层
网络层 网络层 网际层
数据链路层 数据链路层 网络接口层
物理层 物理层 网络接口层

三、TCP/IP四层模型

TCP/IP四层模型是实际应用中最广泛的，包括：应用层、运输层、网际层、网络接口层。

四、数据封装与解封装

数据在网络传输过程中，每经过一层就会加上该层的头部信息，这个过程叫做"封装"。接收端则需要"解封装"。

总结

  • OSI七层模型：理论模型，概念清晰，用于教学和协议设计参考
  • 五层模型：OSI与TCP/IP的综合，业界常用
  • TCP/IP四层模型：实际使用的模型，简化为应用层、运输层、网际层、网络接口层

理解网络分层模型是理解网络协议的基础，每一层都有其特定的功能和协议。
