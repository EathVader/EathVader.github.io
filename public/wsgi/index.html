WSGI 接口详解

来源：廖雪峰 · Flask 源码分析

Web 应用的本质

了解了 HTTP 协议和 HTML 文档后，我们其实就明白了一个 Web 应用的本质：

🌐
浏览器发送 HTTP 请求
→
🖥️
服务器收到请求，生成 HTML
→
📤
服务器发送 HTTP 响应
→
👁️
浏览器渲染页面

最简单的 Web 应用就是把 HTML 文件保存好，用 Apache、Nginx 等静态服务器读取并返回。但如果要动态生成 HTML，就需要自己实现上述步骤。然而，接受 HTTP 请求、解析请求、发送响应都是苦力活——所以我们不自己写底层代码，而是需要一个统一的接口，让我们专心用 Python 编写 Web 业务。

这个接口就是 WSGI：Web Server Gateway Interface。

核心理解： WSGI 是 Web 服务器和 Web 框架（Python）之间的接口规范。框架实现 WSGI 应用，服务器调用 WSGI 应用，两者通过 WSGI 规范解耦。

WSGI 接口定义

WSGI 接口定义非常简单，它只要求 Web 开发者实现一个函数，就可以响应 HTTP 请求：

最简单的 WSGI 应用
def application(environ, start_response):
    start_response('200 OK', [('Content-Type', 'text/html')])
    return '<h1>Hello, web!</h1>'

application() 函数接收两个参数：

参数 类型 说明
environ dict 包含所有 HTTP 请求信息的字典，由 WSGI 服务器提供
start_response callable 发送 HTTP 响应头的函数，只能调用一次

start_response() 接收两个参数：

  • HTTP 响应码：如 '200 OK'
  • HTTP Headers：一组 list，每个 Header 用两个 str 的 tuple 表示

函数的返回值将作为 HTTP 响应的 Body 发送给浏览器。

运行 WSGI 服务

application() 函数必须由 WSGI 服务器来调用。Python 内置了一个 WSGI 服务器 wsgiref，它是纯 Python 编写的参考实现，仅供开发和测试使用。

hello.py —— WSGI 处理函数
def application(environ, start_response):
    start_response('200 OK', [('Content-Type', 'text/html')])
    return '<h1>Hello, %s!</h1>' % (environ['PATH_INFO'][1:] or 'web')
server.py —— WSGI 服务器
from wsgiref.simple_server import make_server
from hello import application

# 创建服务器，IP 为空，端口 8000
httpd = make_server('', 8000, application)
print("Serving HTTP on port 8000...")
httpd.serve_forever()

启动后，打开浏览器访问 http://localhost:8000/，可以看到 Hello, web!。输入 http://localhost:8000/Alice，会显示 Hello, Alice!。

environ 字典详解

environ 是 WSGI 服务器提供的字典，包含所有 HTTP 请求信息。常用字段：

字段 说明 示例值
REQUEST_METHOD HTTP 请求方法 'GET', 'POST'
PATH_INFO URL 路径 '/hello'
QUERY_STRING URL 查询参数 'name=Alice&age=25'
CONTENT_TYPE 请求体类型 'application/json'
CONTENT_LENGTH 请求体长度 '42'
SERVER_NAME 服务器主机名 'localhost'
SERVER_PORT 服务器端口 '8000'
HTTP_HOST Host 头 'localhost:8000'
wsgi.input 请求体文件对象 文件流
wsgi.errors 错误输出流 stderr

从 WSGI 到 Web 框架

无论多么复杂的 Web 应用程序，入口都是一个 WSGI 处理函数。但光靠一个 WSGI 函数来处理还是太底层了，我们需要在 WSGI 之上再抽象出 Web 框架，进一步简化 Web 开发。

1

原始 WSGI

手动解析 environ['PATH_INFO']，手动构造 HTML

def app(environ, start_response):
    path = environ['PATH_INFO']
    if path == '/':
        return ['<h1>Home</h1>']
    elif path == '/about':
        return ['<h1>About</h1>']
    start_response('404 Not Found', [])
    return ['Not Found']
2

简单路由

用字典映射 URL 到处理函数

routes = {
    '/': home_handler,
    '/about': about_handler,
}
def app(environ, start_response):
    handler = routes.get(environ['PATH_INFO'])
    if handler:
        return handler(environ, start_response)
    start_response('404', [])
    return ['Not Found']
3

Flask 框架

使用装饰器声明路由，框架自动处理 WSGI 细节

from flask import Flask
app = Flask(__name__)

@app.route("/")
def hello():
    return "Hello World!"

@app.route("/<name>")
def greet(name):
    return f"Hello, {name}!"
Flask 的本质： Flask 框架内部实现了 WSGI 接口，@app.route() 装饰器把函数注册到路由表中。当 WSGI 服务器调用 Flask 的 application(environ, start_response) 时，Flask 根据 URL 查找对应的处理函数并调用。

常见 WSGI 服务器

服务器 特点 适用场景
wsgiref Python 内置，纯 Python 实现 开发测试，不用于生产
Gunicorn 预 fork worker 模型，稳定 生产环境，Linux 首选
uWSGI 功能丰富，性能高 生产环境，功能全面
Waitress 纯 Python，跨平台 Windows 生产环境
mod_wsgi Apache 模块 Apache 部署

总结

  • WSGI 是 Web 服务器和 Web 框架之间的接口规范（PEP 3333）
  • WSGI 应用是一个函数：application(environ, start_response) → body
  • environ 包含所有 HTTP 请求信息，start_response 发送响应头
  • 无论多复杂的 Web 应用，入口都是一个 WSGI 处理函数
  • Web 框架（Flask/Django）在 WSGI 之上抽象出路由、模板、ORM 等功能
  • 生产环境使用 Gunicorn/uWSGI，开发测试使用 wsgiref
