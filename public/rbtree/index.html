1 二叉查找树 / Binary Search Tree

📖

类比 / Analogy

BST 就像一本按字母排序的通讯录：翻到中间看一眼，比目标大就往前翻，比目标小就往后翻。每次排除一半，很快就能找到。

📐 BST 的性质
  • 左子树所有节点的值 < 根节点
  • 右子树所有节点的值 > 根节点
  • 左右子树也分别是 BST
查找 10：9→右→13→左→11→左→10 ✅

2 BST 的退化问题 / The Degeneration Problem

如果按顺序插入 9, 8, 7, 6, 5, 4, 3，BST 会退化成一条链表，查找变成 O(n)！

红黑树通过旋转和变色保持平衡，避免退化
❓ 核心问题

如何让 BST 在插入/删除后自动保持平衡？答案就是红黑树——通过给节点染色 + 旋转操作，保证树高始终为 O(log n)。

3 红黑树的 5 条规则 / Five Rules

🏛️

类比 / Analogy

红黑树就像一个有严格着装规定的公司：每个员工（节点）必须穿红色或黑色衣服，而且有 5 条规定确保"组织架构"不会太偏。

#规则直觉理解
1每个节点是红色或黑色二选一，没有其他颜色
2根节点是黑色老板穿黑色
3叶节点（NIL）是黑色空位算黑色
4红色节点的两个孩子必须是黑色🔴 不能连续出现两个红色（"不能红红相连"）
5从任一节点到其叶节点的所有路径包含相同数目的黑色节点每条路径的"黑色深度"相同（"黑色平衡"）
验证：根是黑色 ✅ 红色节点的孩子都是黑色 ✅ 每条路径黑色节点数 = 2 ✅
🎯 为什么这 5 条规则能保证平衡？

规则 4（不能红红相连）+ 规则 5（黑色平衡）共同约束了树的高度：最长路径（红黑交替）最多是最短路径（全黑）的 2 倍。因此树高 ≤ 2·log(n+1)，保证 O(log n)。

4 旋转操作 / Rotation

旋转是红黑树维持平衡的核心操作，分为左旋和右旋。旋转不改变 BST 的搜索性质，只改变树的形状。

🎡

类比 / Analogy

左旋就像把右孩子"提上来"当新的根，原来的根变成左孩子。就像跷跷板——右边太重了，就把右边提上去。

左旋 / Left Rotation

左旋：Y 上升为根，X 变为 Y 的左孩子，b 从 Y 的左孩子变为 X 的右孩子

右旋 / Right Rotation

右旋：X 上升为根，Y 变为 X 的右孩子，b 从 X 的右孩子变为 Y 的左孩子
💡 旋转的关键性质

旋转前后，中序遍历的结果不变（a < X < b < Y < c），所以 BST 的搜索性质被保持。旋转只改变树的"形状"，不改变"内容"。

5 插入与修复 / Insertion & Fix-up

插入步骤：① 按 BST 规则插入 ② 新节点染红色 ③ 修复可能违反的规则

❓ 为什么新节点染红色？

染红色不会违反规则 5（黑色平衡），最多只违反规则 4（红红相连）。只需要局部修复，比染黑色简单得多。

6 删除与修复 / Deletion & Fix-up

删除比插入更复杂。基本思路：先按 BST 规则删除，如果删除的是黑色节点，就需要修复黑色平衡。

⚠️ 为什么删除红色节点不需要修复？

删除红色节点不影响规则 5（黑色平衡），也不会产生红红相连。所以只有删除黑色节点才需要修复。

🔑 删除修复的核心思想

被删节点留下一个"额外的黑色"。修复过程就是把这个额外黑色向上传递或通过旋转+变色消化掉，直到：

  • 遇到红色节点 → 直接染黑，吸收额外黑色 ✅
  • 到达根节点 → 额外黑色自然消失 ✅
  • 通过 Case 4 的旋转+变色一步到位 ✅

9 2-3-4 树视角 / 2-3-4 Tree Equivalence

红黑树的 5 条规则看起来很"人为"，但如果你知道红黑树本质上是 2-3-4 树的二叉表示，一切就自然了。

🔄

等价关系 / Equivalence

把红色节点"合并"到它的黑色父节点中，红黑树就变成了 2-3-4 树：

红色节点 = 被"拆开"的多路节点。规则 4（不能红红相连）= 不能有 5-节点
红黑树规则2-3-4 树的自然推论
规则 4：不能红红相连2-3-4 树最多 4-节点，不存在 5-节点
规则 5：黑色平衡2-3-4 树所有叶节点在同一层
插入 Case 1：叔叔红 → 变色4-节点分裂，中间键上提
插入 Case 3：旋转3-节点吸收新键
🎯 理解了 2-3-4 树，红黑树的规则就不用死记

红黑树只是 2-3-4 树的一种"编码方式"——用红色边表示同一个多路节点内的连接。所有看似复杂的旋转和变色，本质上都是 2-3-4 树的节点分裂和合并。

10 总结 / Summary

操作时间复杂度最多旋转次数
查找O(log n)0
插入O(log n)2
删除O(log n)3

红黑树 vs AVL 树

红黑树AVL 树
平衡条件黑色平衡（宽松）高度差 ≤ 1（严格）
查找稍慢（树更高）更快（更矮）
插入/删除更快（旋转更少）稍慢（旋转更多）
适用场景写多读少（如 Linux 内核、Java TreeMap）读多写少（如数据库索引）
🌍 红黑树在哪里被使用？
  • Java：TreeMap, TreeSet, HashMap（链表长度 > 8 时转红黑树）
  • C++ STL：std::map, std::set
  • Linux 内核：进程调度 CFS、内存管理 vm_area_struct
  • Nginx：定时器管理
