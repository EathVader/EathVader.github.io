1super() 是什么？

super() 返回一个代理对象，用于调用父类的方法。它不是简单的"调用父类"，而是根据 MRO（方法解析顺序） 查找下一个类。

Python
class Animal:
    def speak(self):
        return "..."

class Dog(Animal):
    def speak(self):
        return super().speak() + " Woof!"

dog = Dog()
print(dog.speak())  # "... Woof!"
💡 关键理解

super() 不是"调用父类"，而是"调用 MRO 中的下一个类"。在单继承中，下一个类就是父类；但在多继承中，下一个类可能不是直接的父类。

2单继承中的 super()

单继承中，super() 的行为很直观：

Python
class A:
    def __init__(self):
        self.a = "A"
        print(f"A.__init__ called, a={self.a}")

class B(A):
    def __init__(self):
        self.b = "B"
        super().__init__()  # 调用 A.__init__
        print(f"B.__init__ called, b={self.b}")

class C(B):
    def __init__(self):
        self.c = "C"
        super().__init__()  # 调用 B.__init__
        print(f"C.__init__ called, c={self.c}")

c = C()
# 输出顺序：
# A.__init__ called, a=A
# B.__init__ called, b=B
# C.__init__ called, c=C

print(C.__mro__)
# (C, B, A, object)
C.__init__
self.c = "C"
→ super()
B.__init__
self.b = "B"
→ super()
A.__init__
self.a = "A"
→ super()
object.__init__
空操作

3多继承中的 super()

多继承中，super() 的行为就不那么直观了：

Python
class A:
    def __init__(self):
        print("A.__init__")
        super().__init__()

class B(A):
    def __init__(self):
        print("B.__init__")
        super().__init__()

class C(A):
    def __init__(self):
        print("C.__init__")
        super().__init__()

class D(B, C):
    def __init__(self):
        print("D.__init__")
        super().__init__()

d = D()
# 输出：
# D.__init__
# B.__init__
# C.__init__
# A.__init__

print(D.__mro__)
# (D, B, C, A, object)
⚠️ 注意

在 D(B, C) 中，super() 的调用顺序是 D → B → C → A，而不是 D → B → A 和 D → C → A。这是因为 super() 遵循 MRO 顺序，每个类只被调用一次。

super() 按照 MRO 顺序依次调用：D → B → C → A → object

4super() 与 MRO 的关系

super() 的本质是根据 MRO 和当前类 找到下一个类：

Python
class A: pass
class B(A): pass
class C(A): pass
class D(B, C): pass

# super() 在 B 中调用时，查找 MRO 中 B 的下一个类
# D.__mro__ = (D, B, C, A, object)
# B 的下一个是 C，不是 A！

class B(A):
    def method(self):
        print(f"B.mro = {B.__mro__}")
        print(f"super() in B → {super().__thisclass__}")
        # super() 会调用 C.method()，不是 A.method()！
🔑 核心原理

super() 接收两个隐式参数：__class__（当前类）和 self（实例）。它从 type(self).__mro__ 中找到 __class__ 的位置，然后返回下一个类的代理对象。

5常见误区

误区一：super() 就是调用父类

Python
# ❌ 错误理解
class Child(Parent):
    def method(self):
        super().method()  # 不是调用 Parent.method()！

# ✅ 正确理解：调用 MRO 中 Child 的下一个类的方法

误区二：super() 只能用于单继承

Python
# ✅ super() 在多继承中同样有效
class D(B, C):
    def __init__(self):
        super().__init__()  # 调用 B.__init__，B 的 super() 会调用 C.__init__

误区三：父类不需要调用 super()

Python
# ❌ 如果 A 不调用 super()，MRO 链会断裂
class A:
    def __init__(self):
        print("A")
        # 没有 super().__init__() → C.__init__ 不会被调用！

# ✅ 所有参与协作多继承的类都应该调用 super()
class A:
    def __init__(self):
        print("A")
        super().__init__()  # 确保 MRO 链完整
🚨 MRO 链断裂

如果 MRO 中某个类没有调用 super()，它后面的类的方法将不会被调用。这会导致难以发现的 Bug。

6协作式多继承

为了让 super() 在多继承中正常工作，所有类需要协作：

Python
class Drawable:
    def __init__(self, **kwargs):
        super().__init__(**kwargs)
        self.color = kwargs.get('color', 'black')

class Movable:
    def __init__(self, **kwargs):
        super().__init__(**kwargs)
        self.speed = kwargs.get('speed', 0)

class Sprite(Drawable, Movable):
    def __init__(self, **kwargs):
        super().__init__(**kwargs)
        self.name = kwargs.get('name', 'sprite')

s = Sprite(color='red', speed=10, name='hero')
print(s.color)   # 'red'
print(s.speed)   # 10
print(s.name)    # 'hero'
✅ 协作式多继承的要点
  • 所有类都使用 **kwargs 传递参数
  • 所有类都调用 super().__init__(**kwargs)
  • 每个类只处理自己关心的参数
  • 最终基类（如 object）不接收额外参数

7最佳实践

场景推荐做法
单继承使用 super().__init__() 调用父类
多继承所有类都使用 super() 和 **kwargs
参数传递使用 **kwargs 而非位置参数
避免直接调用 ParentClass.method(self)
调试使用 ClassName.__mro__ 查看解析顺序
Python（推荐写法）
class Base:
    def __init__(self, **kwargs):
        super().__init__(**kwargs)

class Mixin1(Base):
    def __init__(self, **kwargs):
        self.x = kwargs.pop('x', 0)
        super().__init__(**kwargs)

class Mixin2(Base):
    def __init__(self, **kwargs):
        self.y = kwargs.pop('y', 0)
        super().__init__(**kwargs)

class Child(Mixin1, Mixin2):
    def __init__(self, **kwargs):
        self.z = kwargs.pop('z', 0)
        super().__init__(**kwargs)

8总结

✅ 核心要点
  • super() 不是"调用父类"，而是"调用 MRO 中的下一个类"
  • 多继承中，super() 遵循 C3 线性化算法确定的 MRO 顺序
  • 协作式多继承需要所有类都调用 super() 并使用 **kwargs
  • 避免直接调用 ParentClass.method(self)，这会破坏 MRO 链
  • 使用 ClassName.__mro__ 调试继承问题
