Flask 0.1 源代码分析

来源：Flask 源码分析

为什么读 Flask 源码？

Flask 框架饱受好评的原因就是因为简洁、明快而且够 Pythonic。Flask 0.1 的代码量只有不到 700 行，但读懂它并不容易。在此之前还需要 WSGI 的一些基础知识。

Flask 的核心理念： 用最少的代码实现最强大的功能。理解 Flask 源码就是理解 Python Web 框架的原理。

Flask 快速上手

通过 Flask 框架，我们可以很快地写一个网页：

最简单的 Flask 应用
from flask import Flask
app = Flask(__name__)

@app.route("/")
def hello():
    return "Hello World!"

if __name__ == "__main__":
    app.run()

就这么简单！@app.route() 装饰器把函数注册到路由表中，app.run() 启动内置服务器。

WSGI 接口规范

WSGI 是一种规范，它定义了 Web App 和 Web 服务器的接口。WSGI 层和 Web 框架层可以看为一层——即在 Web 框架中实现 WSGI 面向 Web 服务器的接口。

PEP 3333 中的 WSGI 应用示例
def simple_app(environ, start_response):
    """Simplest possible application object"""
    status = '200 OK'
    response_headers = [('Content-type', 'text/plain')]
    start_response(status, response_headers)
    return ['Hello world!\n']

这里不用关心 environ 和 start_response，它们由 Web 服务器提供。框架需要为服务器提供标准的接口。

Flask 源码核心概念

1. Flask 应用对象

Flask(__name__) 创建了一个 WSGI 应用实例。Flask 类实现了 __call__ 方法，使其可以被 WSGI 服务器直接调用：

class Flask:
    def __call__(self, environ, start_response):
        return self.wsgi_app(environ, start_response)

2. 路由系统

@app.route() 装饰器把 URL 映射到处理函数：

@app.route("/")
def hello():
    return "Hello World!"

# 等价于
app.add_url_rule("/", "hello", hello)

3. 请求上下文

Flask 使用 request 对象封装 environ 字典，让开发者不用直接操作 WSGI 细节：

from flask import request

@app.route("/hello")
def hello():
    name = request.args.get('name', 'World')
    return f"Hello, {name}!"

Web 服务器层

作为服务器端也有必要提供 WSGI 的服务器端接口。最简单的 WSGI 服务器实现：

WSGI 服务器参考实现
import os, sys

def run_with_cgi(application):
    environ = dict(os.environ.items())
    environ['wsgi.input'] = sys.stdin
    environ['wsgi.errors'] = sys.stderr
    environ['wsgi.version'] = (1, 0)
    
    headers_set = []
    
    def start_response(status, response_headers, exc_info=None):
        headers_set[:] = [status, response_headers]
        return write
    
    result = application(environ, start_response)
    try:
        for data in result:
            if data:
                write(data)
    finally:
        if hasattr(result, 'close'):
            result.close()

通过 WSGI，Web 服务器和框架之间可以连接起来了。

总结

  • WSGI 是服务器和框架之间的接口规范
  • Flask 实现了 WSGI 接口，并在此之上提供了路由、模板、请求封装等功能
  • Flask 0.1 只有不到 700 行代码，但设计精巧
  • 理解 WSGI 是理解 Python Web 框架的基础
