1 问题背景 / Problem Background

🃏

生活类比 / Real-life Analogy

多路归并就像合并多个已排序的扑克牌堆。想象桌上有 k 堆牌，每堆已经从小到大排好。你要把它们合并成一堆有序的牌。最简单的策略：每次看所有牌堆最上面的牌，取最小的那张放到输出堆。重复直到所有牌堆都空了。

K-way merge is like merging multiple sorted piles of playing cards. Look at the top card of each pile, pick the smallest, repeat.

面试经典题：给你 10GB 的无序数据，但只有 100MB 内存，如何排序？

普通排序算法（快排、归并）需要把所有数据加载到内存。当数据量远超内存时，我们需要外部排序（External Sort）——利用磁盘作为辅助存储。

🎯 核心思想
分而治之：先把大文件切成内存能装下的小块，各自排序（Phase 1），再用多路归并把所有有序小块合并成一个有序大文件（Phase 2）。

关键优势：内存消耗与数据总量无关，只与归并路数 k 成正比。

应用场景 / Use Cases

场景说明
数据库排序MySQL ORDER BY 数据量超过 sort_buffer_size 时触发外部排序
LevelDB / RocksDBMajor Compaction 时对多个 SSTable 做多路归并
MapReduceReduce 阶段合并来自多个 Mapper 的有序输出
大文件排序Linux sort 命令处理超大文件时的底层算法

2 算法概览 / Algorithm Overview

外部归并排序分为两个阶段：

Phase 1: 生成归并段 (Run Generation)
1. 从大文件中读取一块数据（内存能容纳的大小）
2. 在内存中用快排/堆排序排好
3. 写回磁盘，形成一个有序归并段（sorted run）
4. 重复直到大文件读完 → 得到 N 个有序归并段
Phase 2: K 路归并 (K-Way Merge)
1. 同时打开 k 个归并段文件
2. 用最小堆（Min-Heap）维护每个文件的当前最小元素
3. 每次从堆顶弹出全局最小值，写入输出文件
4. 从被弹出元素所在的文件读取下一个元素，压入堆
5. 重复直到所有文件读完

复杂度分析 / Complexity

指标值说明
时间复杂度O(N log k)N 个元素，每个元素经过 log k 的堆操作
空间复杂度O(k)堆中最多 k 个元素 + k 个输入缓冲区
I/O 复杂度O(N/B)B 为缓冲区大小，顺序读写

3 Phase 1: 生成归并段 / Run Generation

这一步很直观：把大象切成小块，每块在内存里排好序。

import os, heapq, struct

def generate_sorted_runs(input_file, mem_limit, run_dir='runs'):
    """Phase 1: Split large file into sorted runs."""
    os.makedirs(run_dir, exist_ok=True)
    run_id = 0
    chunk = []
    chunk_size = 0

    with open(input_file, 'r') as f:
        for line in f:
            num = int(line.strip())
            chunk.append(num)
            chunk_size += 1
            # When chunk fills memory, sort and flush
            if chunk_size >= mem_limit:
                chunk.sort()  # In-memory sort (Timsort)
                run_path = os.path.join(run_dir, f'run_{run_id}.txt')
                with open(run_path, 'w') as rf:
                    for v in chunk:
                        rf.write(f'{v}\n')
                chunk = []
                chunk_size = 0
                run_id += 1

    # Flush remaining data
    if chunk:
        chunk.sort()
        run_path = os.path.join(run_dir, f'run_{run_id}.txt')
        with open(run_path, 'w') as rf:
            for v in chunk:
                rf.write(f'{v}\n')
        run_id += 1

    return run_id  # Number of runs generated
⚡ 优化：替换排序 (Replacement Selection)
经典方法每个 run 大小 = 内存大小 M。但用替换排序（基于堆），平均可以生成 2M 大小的 run，减少归并段数量。
思路：维护一个堆，不断读入新元素。如果新元素 ≥ 刚输出的值，加入当前 run；否则标记为"下一轮"。

4 Phase 2: K 路归并 / K-Way Merge

算法步骤 / Algorithm Steps

  1. 从每个归并段读取第一个元素，放入最小堆
  2. 弹出堆顶（全局最小值），写入输出
  3. 从被弹出元素所在的归并段读取下一个元素，压入堆
  4. 如果该归并段已读完，不压入新元素（堆自然缩小）
  5. 重复 2-4 直到堆为空
🔑 关键洞察
堆中永远不会有同一个文件的两个元素。这保证了：
• 堆大小始终 ≤ k（归并路数）
• 每次堆操作 O(log k)，与数据总量无关
• 不会遗漏任何文件的数据 
def k_way_merge(run_dir, num_runs, output_file):
    """Phase 2: Merge sorted runs using a min-heap."""
    # Open all run files
    readers = []
    for i in range(num_runs):
        f = open(os.path.join(run_dir, f'run_{i}.txt'), 'r')
        readers.append(f)

    # Initialize heap: (value, run_index)
    heap = []
    for i, f in enumerate(readers):
        line = f.readline().strip()
        if line:
            heapq.heappush(heap, (int(line), i))

    # Merge
    with open(output_file, 'w') as out:
        while heap:
            val, run_idx = heapq.heappop(heap)
            out.write(f'{val}\n')
            # Read next element from the same run
            line = readers[run_idx].readline().strip()
            if line:
                heapq.heappush(heap, (int(line), run_idx))

    for f in readers:
        f.close()

5 最小堆详解 / Min-Heap Deep Dive

为什么用堆而不是有序数组？因为堆的插入和删除都是 O(log k)，而有序数组插入需要 O(k) 的移动操作。

数据结构取最小值插入删除最小值适合场景
有序数组O(1)O(k) 🐌O(k)k 很小时
最小堆O(1)O(log k) ✅O(log k) ✅k 较大时（推荐）
BST / TreeSetO(log k)O(log k)O(log k)需要更多操作时

Python heapq 实现

Python 标准库 heapq 提供了最小堆实现，heapq.merge() 甚至直接支持多路归并：

import heapq

# heapq.merge 直接做 k 路归并（惰性迭代器）
runs = [
    [2, 7, 14, 38],
    [5, 9, 27, 46],
    [3, 22, 31, 63],
]

# One-liner k-way merge!
merged = list(heapq.merge(*runs))
print(merged)
# [2, 3, 5, 7, 9, 14, 22, 27, 31, 38, 46, 63]
💡 heapq.merge 的内部实现
heapq.merge() 内部就是用最小堆做 k 路归并，和我们手写的逻辑完全一样。
它返回一个惰性迭代器，不会一次性加载所有数据到内存——非常适合处理大文件。

6 I/O 优化 / I/O Optimization

外部排序的瓶颈不是 CPU，而是磁盘 I/O。优化策略围绕"减少 I/O 次数"和"让每次 I/O 更高效"展开。

6.1 缓冲区策略 / Buffer Strategy

📦 内存分配公式
设总内存为 M，归并路数为 k：
• 每个输入缓冲区大小 = M / (k + 1)
• 输出缓冲区大小 = M / (k + 1)
• 堆只需 k 个元素，内存可忽略

权衡：k 越大 → 缓冲区越小 → I/O 次数越多；k 越小 → 可能需要多轮归并。

6.2 多轮归并 / Multi-Pass Merge

如果归并段数量 N 远大于 k，一轮归并不够，需要多轮：

# 例: 1000 个归并段, k=100
# Round 1: 1000 个 run → 合并为 10 个 run (每次合并 100 个)
# Round 2: 10 个 run → 合并为 1 个 run (最终结果)
# 总轮数 = ceil(log_k(N)) = ceil(log_100(1000)) = 2 轮

import math
def num_passes(num_runs, k):
    """Calculate number of merge passes needed."""
    if num_runs <= 1:
        return 0
    return math.ceil(math.log(num_runs) / math.log(k))

6.3 优化技巧汇总

技巧效果原理
增大缓冲区减少 I/O 次数每次读/写更多数据，摊薄寻道开销
顺序读写利用磁盘预读顺序 I/O 比随机 I/O 快 100 倍
增大 k减少归并轮数log_k(N) 随 k 增大而减小
替换排序减少归并段数平均生成 2M 大小的 run
异步 I/OCPU 和 I/O 并行读下一批数据时 CPU 处理当前批
败者树减少比较次数比最小堆少约 50% 的比较
🏆 败者树 vs 最小堆
败者树（Loser Tree）是多路归并的另一种数据结构，每次取最小值只需 log k 次比较（堆需要 2 log k 次）。
在 k 很大时（如 k=1000），败者树的性能优势明显。Linux sort 命令内部就使用败者树。

7 完整代码 / Complete Code

以下是完整的 Python 实现，可以直接运行：

"""
External Merge Sort - Complete Implementation
磁盘多路归并排序完整实现
"""
import os
import heapq
import random
import tempfile
import time


def generate_test_file(path, n, max_val=10_000_000):
    """Generate a test file with n random integers."""
    with open(path, 'w') as f:
        for _ in range(n):
            f.write(f'{random.randint(0, max_val)}\n')


def external_sort(input_file, output_file, mem_limit=1000):
    """
    External merge sort.

    Args:
        input_file: Path to unsorted input file (one integer per line)
        output_file: Path to write sorted output
        mem_limit: Max number of elements to hold in memory at once
    """
    run_dir = tempfile.mkdtemp(prefix='ext_sort_')

    # Phase 1: Generate sorted runs
    num_runs = _generate_runs(input_file, run_dir, mem_limit)
    print(f'Phase 1 done: {num_runs} sorted runs generated')

    # Phase 2: K-way merge
    _k_way_merge(run_dir, num_runs, output_file)
    print(f'Phase 2 done: merged into {output_file}')

    # Cleanup temp files
    for f in os.listdir(run_dir):
        os.remove(os.path.join(run_dir, f))
    os.rmdir(run_dir)


def _generate_runs(input_file, run_dir, mem_limit):
    """Phase 1: Read chunks, sort in memory, write sorted runs."""
    run_id = 0
    chunk = []

    with open(input_file, 'r') as f:
        for line in f:
            chunk.append(int(line.strip()))
            if len(chunk) >= mem_limit:
                chunk.sort()
                _write_run(run_dir, run_id, chunk)
                chunk = []
                run_id += 1

    if chunk:
        chunk.sort()
        _write_run(run_dir, run_id, chunk)
        run_id += 1

    return run_id


def _write_run(run_dir, run_id, data):
    """Write a sorted run to disk."""
    path = os.path.join(run_dir, f'run_{run_id}.txt')
    with open(path, 'w') as f:
        for v in data:
            f.write(f'{v}\n')


def _k_way_merge(run_dir, num_runs, output_file):
    """Phase 2: Merge all sorted runs using a min-heap."""
    readers = []
    heap = []

    # Open all runs and seed the heap
    for i in range(num_runs):
        f = open(os.path.join(run_dir, f'run_{i}.txt'), 'r')
        readers.append(f)
        line = f.readline().strip()
        if line:
            heapq.heappush(heap, (int(line), i))

    # Merge loop
    with open(output_file, 'w') as out:
        while heap:
            val, run_idx = heapq.heappop(heap)
            out.write(f'{val}\n')
            line = readers[run_idx].readline().strip()
            if line:
                heapq.heappush(heap, (int(line), run_idx))

    for f in readers:
        f.close()


def verify_sorted(path):
    """Verify that a file is sorted."""
    prev = float('-inf')
    count = 0
    with open(path, 'r') as f:
        for line in f:
            val = int(line.strip())
            if val < prev:
                return False, count
            prev = val
            count += 1
    return True, count


if __name__ == '__main__':
    input_path = 'test_input.txt'
    output_path = 'test_output.txt'
    n = 100_000

    print(f'Generating {n} random numbers...')
    generate_test_file(input_path, n)

    start = time.time()
    external_sort(input_path, output_path, mem_limit=1000)
    elapsed = time.time() - start

    ok, count = verify_sorted(output_path)
    print(f'Sorted {count} numbers in {elapsed:.2f}s')
    print(f'Verification: {"PASS ✅" if ok else "FAIL ❌"}')
