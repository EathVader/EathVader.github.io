1 什么是 Radix Tree / What is Radix Tree

📁

类比 / Analogy

普通 Trie 树就像文件路径 /u/s/r/l/o/c/a/l，每个字符一层目录。Radix Tree 则像 /usr/local，把没有分支的路径合并成一个节点。目录更少，查找更快。

📐 Radix Tree = 压缩 Trie

Radix Tree（又叫 Patricia Trie、Compact Trie）是 Trie 的空间优化版本。核心思想：将只有一个子节点的链式节点合并为一个节点，边上存储字符串而非单个字符。

3 压缩过程 / Compression

Radix Tree 有两种压缩方式：

A. 尾部压缩 / Tail Compression

将没有分支的"尾巴"合并为一个节点。例如 t→o→a→s→t 压缩为 toast。

B. 路径压缩 / Path Compression

将所有没有分支的链式节点合并。这是更彻底的压缩，也是 Radix Tree 的标准形式。

🔑 压缩规则

如果一个内部节点只有一个子节点，就把它和子节点合并。合并后，边上存储的是字符串而非单个字符。重复此过程直到每个内部节点都有 ≥2 个子节点（或者是单词结尾）。

查找时有三种情况：

  1. 公共前缀为空：当前边不匹配，尝试兄弟边
  2. 公共前缀 = 边标签：完全匹配，消耗后继续向下
  3. 公共前缀 < 边标签：部分匹配，查找失败

插入 / Insert

插入时如果发现部分匹配（公共前缀 < 边标签），需要分裂节点。

删除 / Delete

删除后如果某个内部节点只剩一个子节点，需要合并节点（压缩的逆操作的逆操作）。

5 节点分裂 / Node Splitting

插入 "team" 到已有 "test" 的树中：

// 节点分裂伪代码
function split(node, prefixLen):
    // 1. 创建新的中间节点，标签 = 原标签的前 prefixLen 个字符
    mid = new Node(label = node.label[0:prefixLen])
    // 2. 原节点标签截断为剩余部分
    node.label = node.label[prefixLen:]
    // 3. 中间节点的子节点 = 原节点
    mid.children[node.label[0]] = node
    // 4. 返回中间节点（替换原节点在父节点中的位置）
    return mid
⚠️ 分裂是 Radix Tree 的核心难点

每次插入最多触发一次分裂。分裂后树仍然保持压缩状态。删除时可能需要反向操作——合并只有一个子节点的内部节点。

6 代码实现 / Implementation

class RadixNode {
    String label;
    Map<Character, RadixNode> children = new HashMap<>();
    boolean isEnd;

    RadixNode(String label, boolean isEnd) {
        this.label = label;
        this.isEnd = isEnd;
    }
}

class RadixTree {
    RadixNode root = new RadixNode("", false);

    void insert(String word) {
        RadixNode node = root;
        while (word.length() > 0) {
            boolean found = false;
            for (var entry : node.children.entrySet()) {
                String key = entry.getValue().label;
                int cp = commonPrefix(key, word);
                if (cp == 0) continue;
                found = true;
                if (cp == key.length()) {
                    // 完全匹配边标签，继续向下
                    node = entry.getValue();
                    word = word.substring(cp);
                } else {
                    // 部分匹配，需要分裂
                    RadixNode child = entry.getValue();
                    RadixNode mid = new RadixNode(key.substring(0, cp), false);
                    child.label = key.substring(cp);
                    mid.children.put(child.label.charAt(0), child);
                    node.children.put(mid.label.charAt(0), mid);
                    node = mid;
                    word = word.substring(cp);
                }
                break;
            }
            if (!found) {
                node.children.put(word.charAt(0),
                    new RadixNode(word, true));
                return;
            }
        }
        node.isEnd = true;
    }

    int commonPrefix(String a, String b) {
        int i = 0;
        while (i < a.length() && i < b.length()
               && a.charAt(i) == b.charAt(i)) i++;
        return i;
    }
}

7 Trie vs Radix Tree

TrieRadix Tree
每条边存储单个字符字符串（多个字符）
节点数量O(所有字符总数)O(字符串个数)
查找时间O(L)O(L)
空间大（大量单子节点）紧凑
实现复杂度简单中等（需处理分裂/合并）
前缀搜索✅✅
💡 何时选 Radix Tree

字符串很长且有大量公共前缀时（如 URL、文件路径、IP 地址），Radix Tree 的空间优势非常明显。节点数从 O(所有字符总数) 降到 O(字符串个数)。

8 实际应用 / Applications

🐧 Linux 内核页缓存

Linux 内核使用 Radix Tree（现已演进为 XArray）管理页缓存（page cache）。每个文件的页面按偏移量索引，Radix Tree 提供 O(log n) 的查找。

// Linux 内核中的 Radix Tree（简化）
// 文件 inode → radix_tree → 页面
struct address_space {
    struct xarray  i_pages;  // 原 radix_tree_root
    ...
};

🌐 HTTP 路由器

Go 语言的 net/http 标准库和 httprouter 等高性能路由器使用 Radix Tree 匹配 URL 路径。

// httprouter 的路由注册
// /api/users/:id  和  /api/users/list
// 共享前缀 "/api/users/" 只存一次
router.GET("/api/users/:id", getUser)
router.GET("/api/users/list", listUsers)

🔤 IP 路由表 (Longest Prefix Match)

网络路由器使用二进制 Radix Tree 进行最长前缀匹配。每个 IP 地址转为二进制后在树中查找，找到最长匹配的路由条目。

📦 其他应用

  • Redis：Cluster 模式的 slot 映射
  • 数据库索引：ART (Adaptive Radix Tree) 用于内存数据库
  • 自动补全：比普通 Trie 更省内存的实现
🚀 ART: Adaptive Radix Tree

2013 年提出的 ART 是 Radix Tree 的现代演进。它根据子节点数量动态选择 4 种节点类型（Node4/16/48/256），在空间和速度之间取得最优平衡。被 HyPer、DuckDB 等内存数据库采用。

9 总结 / Summary

操作时间复杂度说明
查找O(L)L = 字符串长度
插入O(L)最多一次分裂
删除O(L)可能触发合并
空间O(N × L̄)N = 字符串数，L̄ = 平均长度
🎯 一句话总结

Radix Tree = 压缩版 Trie。通过合并无分支路径，用更少的节点存储相同的字符串集合。适合长字符串、大量公共前缀的场景（URL 路由、IP 查找、文件系统）。
