1 什么是 LRU Cache？

🧊

生活类比：冰箱门

想象你的冰箱——最近使用的放门口。每次拿东西出来用完，放回去时总是放在最前面。冰箱满了要放新东西？把最里面（最久没用的）那个扔掉。这就是 LRU（Least Recently Used）的核心思想。

LRU Cache 是一种固定容量的缓存，当容量满时，淘汰最久没有被访问的数据。它需要支持两个操作，且都是 O(1) 时间复杂度：

操作说明复杂度
get(key)查询 key 对应的 value，不存在返回 -1O(1)
put(key, value)插入/更新键值对，满了则淘汰最久未用的O(1)
💡 为什么需要 LRU？

Cache 空间有限，需要淘汰策略。LRU 基于一个假设：最近用到的数据，被再次使用的概率更大。CPU 缓存、操作系统页面置换、数据库查询缓存、Redis 都用到了 LRU 或其变体。

💡 为什么用双向链表？

• 单向链表删除节点需要 O(n) 找前驱
• 双向链表有 prev 指针，删除任意节点只需 O(1)
• 哈希表存的是节点引用，直接定位到链表中的节点，无需遍历

🧩 哨兵节点 HEAD / TAIL

使用虚拟的头尾节点（dummy head/tail），避免处理边界条件（空链表、只有一个节点等）。所有真实数据节点都在 HEAD 和 TAIL 之间。

def get(self, key: int) -> int:
    if key not in self.cache:
        return -1
    node = self.cache[key]
    self._move_to_head(node)
    return node.val

4 Put 操作

插入或更新键值对。分两种情况：

情况 1: key 已存在
更新 value，然后移到头部（和 get 类似）
情况 2: key 不存在
创建新节点，插入头部，加入哈希表。如果超出容量，淘汰尾部节点。
图 3: put(4, "four") — 容量已满，淘汰尾部节点 3，新节点 4 插入头部
def put(self, key: int, value: int) -> None:
    if key in self.cache:
        node = self.cache[key]
        node.val = value
        self._move_to_head(node)
    else:
        node = Node(key, value)
        self.cache[key] = node
        self._add_to_head(node)
        self.size += 1
        if self.size > self.capacity:
            tail = self._remove_tail()
            del self.cache[tail.key]
            self.size -= 1

5 淘汰机制

淘汰发生在 put 新 key 且容量已满时。淘汰的永远是链表尾部（TAIL 前面的节点）——即最久没被访问的。

图 4: 淘汰 — 删除 TAIL 前面的节点 C，同时从哈希表中移除
⚠️ 别忘了同步哈希表

从链表删除节点时，必须同时从哈希表中 del self.cache[node.key]。这就是为什么节点要存 key——删除时需要用 key 去清理哈希表。

链表辅助方法：

def _remove(self, node):
    """Remove a node from the linked list."""
    node.prev.next = node.next
    node.next.prev = node.prev

def _add_to_head(self, node):
    """Insert node right after HEAD sentinel."""
    node.prev = self.head
    node.next = self.head.next
    self.head.next.prev = node
    self.head.next = node

def _move_to_head(self, node):
    """Move existing node to head (most recent)."""
    self._remove(node)
    self._add_to_head(node)

def _remove_tail(self):
    """Remove and return the node before TAIL (least recent)."""
    node = self.tail.prev
    self._remove(node)
    return node

6 完整代码

class Node:
    def __init__(self, key=0, val=0):
        self.key = key
        self.val = val
        self.prev = None
        self.next = None

class LRUCache:
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.size = 0
        self.cache = {}  # key -> Node
        # Sentinel nodes to avoid boundary checks
        self.head = Node()
        self.tail = Node()
        self.head.next = self.tail
        self.tail.prev = self.head

    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1
        node = self.cache[key]
        self._move_to_head(node)
        return node.val

    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            node = self.cache[key]
            node.val = value
            self._move_to_head(node)
        else:
            node = Node(key, value)
            self.cache[key] = node
            self._add_to_head(node)
            self.size += 1
            if self.size > self.capacity:
                tail = self._remove_tail()
                del self.cache[tail.key]
                self.size -= 1

    def _remove(self, node):
        node.prev.next = node.next
        node.next.prev = node.prev

    def _add_to_head(self, node):
        node.prev = self.head
        node.next = self.head.next
        self.head.next.prev = node
        self.head.next = node

    def _move_to_head(self, node):
        self._remove(node)
        self._add_to_head(node)

    def _remove_tail(self):
        node = self.tail.prev
        self._remove(node)
        return node
🐍 Python 捷径：OrderedDict

Python 的 collections.OrderedDict 内部就是哈希表+双向链表，可以一行搞定：

from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity: int):
        self.cap = capacity
        self.d = OrderedDict()

    def get(self, key: int) -> int:
        if key not in self.d:
            return -1
        self.d.move_to_end(key)
        return self.d[key]

    def put(self, key: int, value: int) -> None:
        if key in self.d:
            self.d.move_to_end(key)
        self.d[key] = value
        if len(self.d) > self.cap:
            self.d.popitem(last=False)

8 总结

核心要点

概念说明
数据结构哈希表 + 双向链表，两者配合实现 O(1)
哈希表key → Node 引用，O(1) 定位节点
双向链表维护访问顺序，头部最新，尾部最旧
哨兵节点HEAD/TAIL 避免边界判断
get 命中移到头部，返回 value
put 新 key插入头部，超容量则淘汰尾部
淘汰删除 TAIL.prev，同步清理哈希表
