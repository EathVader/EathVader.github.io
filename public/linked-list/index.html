1 什么是链表 / What is Linked List

🚂

类比 / Analogy

链表就像一列火车：每节车厢（节点）装着货物（数据），并用挂钩（指针）连接下一节车厢。你只能从车头开始，沿着挂钩一节一节往后走，不能直接跳到第 5 节车厢。

📐 链表 vs 数组
操作数组链表
随机访问O(1)O(n)
头部插入/删除O(n)O(1)
中间插入/删除O(n)O(1)*

* 已知前驱节点时

2 反转链表 / Reverse Linked List

🔄

类比 / Analogy

想象你在排队，每个人只记得前面那个人。现在要反转队伍：让每个人转身，改为记住后面那个人。你需要一个一个处理，用一只手（prev）记住已经转好的人，另一只手（curr）指向正在处理的人。

迭代法 / Iterative

prev = null
curr = head
→
保存 next
curr.next = prev
→
prev = curr
curr = next
→
重复直到
curr == null
每一步：保存 next → 反转指针 → 移动 prev 和 curr
// 迭代反转 / Iterative reversal - O(n) time, O(1) space
ListNode reverse(ListNode head) {
    ListNode prev = null, curr = head;
    while (curr != null) {
        ListNode next = curr.next;  // 保存下一个
        curr.next = prev;           // 反转指针
        prev = curr;                // prev 前进
        curr = next;                // curr 前进
    }
    return prev;
}

// 递归反转 / Recursive reversal - O(n) time, O(n) space
ListNode reverseRecursive(ListNode head) {
    if (head == null || head.next == null) return head;
    ListNode newHead = reverseRecursive(head.next);
    head.next.next = head;  // 后继指向自己
    head.next = null;       // 断开原指针
    return newHead;
}
⚡ 复杂度对比

迭代：时间 O(n)，空间 O(1) — 推荐
递归：时间 O(n)，空间 O(n)（调用栈）— 更优雅但有栈溢出风险

3 环检测 / Cycle Detection (Floyd's Algorithm)

🐢🐇

类比 / Analogy

想象一条环形跑道，乌龟每次走 1 步，兔子每次走 2 步。如果跑道有环，兔子一定会从后面追上乌龟（就像操场跑步被套圈）。如果没有环，兔子会先跑到终点（null）。

节点 3→4→5→6→3 形成环，快慢指针必在环内相遇
// Floyd 环检测 / Floyd's cycle detection
boolean hasCycle(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;       // 🐢 走 1 步
        fast = fast.next.next;  // 🐇 走 2 步
        if (slow == fast) return true;  // 相遇 = 有环
    }
    return false;  // fast 到达 null = 无环
}

// 找环入口 / Find cycle entry point
ListNode detectCycle(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) {
            slow = head;  // slow 回到起点
            while (slow != fast) {
                slow = slow.next;  // 同速前进
                fast = fast.next;
            }
            return slow;  // 相遇点 = 环入口
        }
    }
    return null;
}
🧮 为什么 slow 回到 head 后能找到环入口？

设起点到环入口距离为 a，环入口到相遇点距离为 b，环长为 c。相遇时 slow 走了 a+b，fast 走了 a+b+nc。因为 fast = 2×slow，所以 a+b = nc，即 a = nc-b = (n-1)c + (c-b)。从 head 走 a 步 = 从相遇点走 (c-b) + 若干圈，两者在环入口相遇。

4 合并两个有序链表 / Merge Two Sorted Lists

双指针比较，每次取较小值，O(n+m) 时间
// 合并两个有序链表 / Merge two sorted lists
ListNode mergeTwoLists(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(0), curr = dummy;
    while (l1 != null && l2 != null) {
        if (l1.val <= l2.val) { curr.next = l1; l1 = l1.next; }
        else                  { curr.next = l2; l2 = l2.next; }
        curr = curr.next;
    }
    curr.next = (l1 != null) ? l1 : l2;
    return dummy.next;
}

// 链表归并排序 / Merge sort for linked list
ListNode sortList(ListNode head) {
    if (head == null || head.next == null) return head;
    // 快慢指针找中点
    ListNode slow = head, fast = head.next;
    while (fast != null && fast.next != null) {
        slow = slow.next; fast = fast.next.next;
    }
    ListNode mid = slow.next;
    slow.next = null;  // 断开
    return mergeTwoLists(sortList(head), sortList(mid));
}

🎯 链表题通用技巧
  • dummy 节点：简化头节点处理，避免大量 null 判断
  • 快慢指针：找中点、检测环、找倒数第 k 个
  • 画图：指针操作容易出错，一定要画图理清顺序
  • 先保存再修改：修改 next 前先保存原值
