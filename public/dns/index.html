1 为什么需要 DNS？

📖

生活类比：电话簿

DNS 就像电话号码簿：

  • 你记得朋友的名字（域名），但不记得电话号码（IP）
  • 查电话簿 → 名字 → 电话号码
  • DNS → 域名 → IP 地址
💡 核心概念
  • 域名：人类可读的地址，如 www.example.com
  • IP 地址：计算机可读的地址，如 93.184.216.34
  • DNS 服务器：域名到 IP 的映射服务
  • 根域名服务器：.com、.org 等顶级域的入口

2 查询过程

两种查询方式

❶ 客户端

向本地 DNS 服务器发起查询请求 www.example.com

❷ 本地 DNS

缓存未命中，向根服务器查询 .com 的权威服务器

❸ 根服务器

返回 .com 服务器的 IP（不负责具体域名）

❹ .com 服务器

返回 example.com 的权威 DNS 服务器 IP

❺ 权威 DNS

查询 www.example.com → 返回具体 IP

❻ 返回

本地 DNS 返回 IP 给客户端，并缓存结果

💡 实际场景

大多数情况下，客户端到本地 DNS 使用递归查询，本地 DNS 到其他服务器使用迭代查询。

3 DNS 记录类型

类型说明示例
AIPv4 地址example.com → 93.184.216.34
AAAAIPv6 地址example.com → 2606:2800:220:1:248:1893:25c8:1946
CNAME别名www.example.com → example.com
MX邮件服务器@ → mail.example.com
TXT文本记录SPF/DKIM 验证
NS域名服务器example.com → ns1.example.com
SOA起始授权记录域名的管理信息
📝 CNAME vs A 记录
  • A 记录：域名直接指向 IP
  • CNAME：域名指向另一个域名（alias）
  • ⚠️ CNAME 不能和其他记录共存（除了 DNSSEC）

4 缓存机制

DNS 查询涉及多级缓存，大幅减少延迟：

💾

类比：图书馆

  • 浏览器缓存 = 你刚查过的电话本（记得最清楚）
  • 操作系统缓存 = 家里书架上的电话本（记得比较清楚）
  • 本地 DNS = 小区物业的电话本（记得相当清楚）
  • ISP DNS = 市图书馆的电话本（记得最久）

TTL 的影响

⚠️ 注意事项
  • 修改 DNS 后：需要等待 TTL 过期才能生效
  • TTL 设置：修改频繁的域名 TTL 短，稳定的域名 TTL 长
  • 缓存失效：TTL=0 时立即失效，需要重新查询
