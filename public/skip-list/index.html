1 为什么需要跳表 / Why

🚇

类比 / Analogy

普通链表像公交车，每站都停。跳表像地铁：有快线（只停大站）和慢线（每站都停）。找目标时先坐快线跳过大段距离，接近目标后换慢线精确定位。

2 什么是跳表 / What

📐 跳表的结构

跳表是多层有序链表。最底层（L0）包含所有元素，每往上一层，节点数约减半。每个节点随机决定自己出现在几层中。

⚡ 为什么是 O(log n)？

每下降一层，搜索范围大约减半（因为每层约有上层 2 倍的节点）。层数 = O(log n)，每层最多走常数步，总共 O(log n)。和二分查找的原理一样！

5 插入与层高 / Insert

🎲 随机层高（抛硬币法）

插入新节点时，用"抛硬币"决定层高：正面（概率 p=1/2）就加一层，反面就停。平均层高 = 1/(1-p) = 2。这样无需复杂的平衡操作，概率上就能保持平衡。

int randomLevel() {
    int level = 0;
    while (Math.random() < 0.5 && level < MAX_LEVEL)
        level++;
    return level;
}

6 代码实现 / Implementation

class SkipNode {
    int val;
    SkipNode[] next; // next[i] = 第 i 层的下一个节点
    SkipNode(int val, int level) {
        this.val = val;
        this.next = new SkipNode[level + 1];
    }
}

class SkipList {
    int maxLevel = 16, level = 0;
    SkipNode head = new SkipNode(Integer.MIN_VALUE, maxLevel);

    void insert(int val) {
        SkipNode[] update = new SkipNode[maxLevel + 1];
        SkipNode curr = head;
        for (int i = level; i >= 0; i--) {
            while (curr.next[i] != null && curr.next[i].val < val)
                curr = curr.next[i];
            update[i] = curr; // 记录每层的前驱
        }
        int newLevel = randomLevel();
        if (newLevel > level) {
            for (int i = level + 1; i <= newLevel; i++)
                update[i] = head;
            level = newLevel;
        }
        SkipNode node = new SkipNode(val, newLevel);
        for (int i = 0; i <= newLevel; i++) {
            node.next[i] = update[i].next[i]; // 链表插入
            update[i].next[i] = node;
        }
    }

    boolean search(int val) {
        SkipNode curr = head;
        for (int i = level; i >= 0; i--)
            while (curr.next[i] != null && curr.next[i].val < val)
                curr = curr.next[i];
        curr = curr.next[0];
        return curr != null && curr.val == val;
    }
}

8 实际应用 / Applications

🌍 工业级应用
  • Redis：Sorted Set（有序集合）的底层实现就是跳表
  • LevelDB / RocksDB：MemTable 使用跳表存储内存中的有序数据
  • Java ConcurrentSkipListMap：JDK 内置的并发有序 Map
  • Apache Lucene：倒排索引中的 posting list 合并
💡

为什么 Redis 选跳表而不是红黑树？

Redis 作者 antirez 的回答：① 跳表实现更简单，bug 更少 ② 范围查询（ZRANGEBYSCORE）天然高效 ③ 通过调整 p 值可以灵活平衡时间和空间。

9 总结 / Summary
