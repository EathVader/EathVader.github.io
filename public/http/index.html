1 HTTP 是什么？

📬

生活类比：邮递系统

HTTP 就像邮递系统中的信件格式规范：

  • 信封格式（请求行/状态行）
  • 信件内容格式（头信息）
  • 信件正文（请求/响应体）
  • 邮编/地址（URL）

无论你寄什么信，都得按这个格式来，邮递员才能帮你送达。

💡 核心概念
  • HTTP（HyperText Transfer Protocol）：超文本传输协议
  • 客户端-服务器：浏览器发起请求，服务器返回响应
  • 无状态：每个请求都是独立的，服务器不记得之前的请求
  • 明文传输：数据以明文形式发送（ HTTPS 是加密版本）

2 HTTP 请求

HTTP 请求由四部分组成：

请求行

📝 格式
方法 URI HTTP版本

例如：GET /api/users HTTP/1.1

请求头

常见头说明
Host目标服务器域名和端口
User-Agent客户端标识（浏览器信息）
Accept客户端能接受的响应类型
Content-Type请求体的类型
Content-Length请求体的长度
CookieCookie 信息
Authorization认证信息

请求体

GET 请求通常没有请求体，POST/PUT 请求才有。常见格式：

  • application/x-www-form-urlencoded：表单数据
  • application/json：JSON 数据
  • multipart/form-data：文件上传

3 HTTP 响应

HTTP 响应也由四部分组成：

状态行

📝 格式
HTTP版本 状态码 原因短语

例如：HTTP/1.1 404 Not Found

响应头

常见头说明
Content-Type响应体的类型
Content-Length响应体的长度
Server服务器信息
Date响应时间
Set-Cookie设置 Cookie
Cache-Control缓存控制
Location重定向地址

4 请求方法

方法说明幂等
GET获取资源✓
POST创建资源✗
PUT完全替换资源✓
DELETE删除资源✓
PATCH部分更新资源✗
HEAD只获取头部，不获取体✓
OPTIONS查询支持的 HTTP 方法✓
📋

类比：图书馆

  • GET：查书（只看不改）
  • POST：新书入库（创建）
  • PUT：书架整层替换（完全替换）
  • DELETE：销毁图书
  • PATCH：更新书的某一页

常见状态码

状态码说明

301 Moved Permanently永久重定向

7 HTTP 与 TCP/IP 的关系

HTTP 是构建在 TCP/IP 协议之上的，是 TCP/IP 协议族的一个子集。理解它们的关系有助于深入理解 HTTP 的工作原理。

TCP/IP 协议族分层

数据包封装过程

应用程序数据在发送到物理网络之前，会沿着协议栈从上往下传递。每层协议都将在上层数据的基础上加上自己的头部信息：

📦 封装与解封装

发送端：应用层数据 → 加上TCP头 → 加上IP头 → 加上以太网头/尾 → 物理网络

接收端：物理网络 → 去掉以太网头/尾 → 去掉IP头 → 去掉TCP头 → 应用层数据

下层协议的头部信息对上层协议是透明的，应用层只需要关心应用服务的实现。

HTTP 与 TCP/IP、DNS 的关系

🔄 完整通信过程
  1. 客户端通过 DNS 服务查询到域名的 IP 地址
  2. 浏览器生成 HTTP 请求
  3. HTTP 请求通过 TCP/IP 协议发送给 Web 服务器
  4. Web 服务器根据请求生成响应内容
  5. 响应通过 TCP/IP 协议返回给客户端

TCP 三次握手

HTTP 使用 TCP 作为传输协议，TCP 提供面向连接、可靠的传输。建立连接时采用三次握手：

🤝 三次握手过程
  1. 第一次握手：客户端发送 SYN 报文，进入 SYN_SEND 状态
  2. 第二次握手：服务端收到 SYN，回复 SYN+ACK，进入 SYN_RECV 状态
  3. 第三次握手：客户端收到 SYN+ACK，回复 ACK，双方进入 ESTABLISHED 状态

三次握手完成后，TCP 为连接双方维持连接状态，保证数据传输的可靠性。
