GIL：为什么多线程不够用

Python 由于 GIL（全局解释锁） 的存在，多线程并不能充分利用多核处理器。GIL 保证同一时刻只有一个线程在执行 Python 字节码。

结论：如果是 CPU 计算密集型任务，应该使用多进程模块 multiprocessing。它给每个进程赋予单独的 Python 解释器，规避了 GIL 问题。

单进程 vs 多进程性能对比

用计算 Fibonacci 数列来演示性能差异：

Python
import time
import multiprocessing

def profile(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        func(*args, **kwargs)
        end = time.time()
        print('COST: {}'.format(end - start))
    return wrapper

def fib(n):
    if n <= 2:
        return 1
    return fib(n-1) + fib(n-2)

@profile
def no_multiprocess():
    fib(35)
    fib(35)

@profile
def has_multiprocess():
    jobs = []
    for i in range(2):
        p = multiprocessing.Process(target=fib, args=(35,))
        p.start()
        jobs.append(p)
    for p in jobs:
        p.join()

no_multiprocess()
has_multiprocess()
运行结果
COST: 4.66861510277
COST: 2.5424861908

多进程让效率几乎翻倍！但需要注意，这个时间取决于最慢的那个进程的执行时间。

进程池 Pool

使用 Pool 是更优雅的写法：

Python
from multiprocessing import Pool

pool = Pool(2)
pool.map(fib, [35]*2)

map 方法和内置的 map 用法一样，但有多进程支持。

dummy 模块：一套代码，两种模式

multiprocessing.dummy 虽然在多进程模块中，但接口和多线程基本一样：

multiprocessing.dummy replicates the API of multiprocessing but is no more than a wrapper around the threading module.
Python
# CPU 密集型：用多进程
from multiprocessing import Pool

# I/O 密集型：用多线程（dummy）
from multiprocessing.dummy import Pool
经验技巧：如果拿不准任务是 CPU 密集还是 I/O 密集型，分别用两种方式试一下，哪个快用哪个。现在统一直接用多进程，切换也很方便。

基于 Pipe 的 parmap

进程间通信（IPC）常用方式：RPC、Socket、Pipe（管道）、消息队列（Queue）。

基本管道示例

Python
from multiprocessing import Process, Pipe

def f(conn):
    conn.send(['hello'])
    conn.close()

parent_conn, child_conn = Pipe()
p = Process(target=f, args=(child_conn,))
p.start()
print(parent_conn.recv())  # 输出: ['hello']
p.join()

Pipe 返回管道两边的对象：「父连接」和「子连接」。

解决类方法无法 pickle 的问题

当函数放在类里面时，Pool.map 会报错：

Python — PicklingError
PicklingError: Can't pickle : 
attribute lookup __builtin__.instancemethod failed

使用管道可以解决这个问题：

Python — parmap 实现
from multiprocessing import Pool, Process, Pipe
from itertools import izip

def spawn(f):
    def func(pipe, item):
        pipe.send(f(item))
        pipe.close()
    return func

def parmap(f, items):
    pipe = [Pipe() for _ in items]
    proc = [Process(target=spawn(f),
                    args=(child, item))
            for item, (parent, child) in izip(items, pipe)]
    [p.start() for p in proc]
    [p.join() for p in proc]
    return [parent.recv() for (parent, child) in pipe]

class CalculateFib(object):
    @classmethod
    def fib(cls, n):
        if n <= 2:
            return 1
        return cls.fib(n-1) + cls.fib(n-2)
    
    def parmap_run(self):
        print(parmap(self.fib, [35]*2))

缺点：每个任务创建一个进程，没有使用进程池。

基于 Queue 的改进版 parmap

使用队列可以支持指定进程池大小：

Python
from multiprocessing import Queue, Process, cpu_count

def apply_func(f, q_in, q_out):
    while not q_in.empty():
        i, item = q_in.get()
        q_out.put((i, f(item)))

def parmap(f, items, nprocs=cpu_count()):
    q_in, q_out = Queue(), Queue()
    proc = [Process(target=apply_func, args=(f, q_in, q_out))
            for _ in range(nprocs)]
    sent = [q_in.put((i, item)) for i, item in enumerate(items)]
    [p.start() for p in proc]
    res = [q_out.get() for _ in sent]
    [p.join() for p in proc]
    return [item for _, item in sorted(res)]

使用 enumerate 保留任务顺序，最后排序恢复。

生产者/消费者模型

使用两个队列：一个存储待完成任务，一个存储结果：

Python
from multiprocessing import Process, JoinableQueue, Queue
from random import random
import time

tasks_queue = JoinableQueue()
results_queue = Queue()

def producer(in_queue):
    while 1:
        wt = random()
        time.sleep(wt)
        in_queue.put((double, wt))
        if wt > 0.9:
            in_queue.put(None)  # 结束信号
            break

def consumer(in_queue, out_queue):
    while 1:
        task = in_queue.get()
        if task is None:
            break
        func, arg = task
        result = func(arg)
        in_queue.task_done()
        out_queue.put(result)
注意：进程的 Queue 类不支持 task_done 和 join 方法，需要使用 JoinableQueue。

进程间共享状态

共享内存（Value / Array）

Python
from multiprocessing import Process, Lock
from multiprocessing.sharedctypes import Value, Array
from ctypes import Structure, c_bool, c_double

lock = Lock()

class Point(Structure):
    _fields_ = [('x', c_double), ('y', c_double)]

def modify(n, b, s, arr, A):
    n.value **= 2
    b.value = True
    s.value = s.value.upper()
    arr[0] = 10
    for a in A:
        a.x **= 2
        a.y **= 2

n = Value('i', 7)
b = Value(c_bool, False, lock=False)
s = Array('c', 'hello world', lock=lock)
arr = Array('i', range(5), lock=True)
A = Array(Point, [(1.875, -6.25), (-5.75, 2.0)], lock=lock)

p = Process(target=modify, args=(n, b, s, arr, A))
p.start()
p.join()

print(n.value)        # 49
print(b.value)        # True
print(s.value)        # HELLO WORLD
print(arr[:])         # [10, 1, 2, 3, 4]
print([(a.x, a.y) for a in A])  # [(3.515625, 39.0625), (33.0625, 4.0)]

服务器进程（Manager）

Manager 对象控制一个服务器进程，其他进程通过代理访问：

Python
from multiprocessing import Manager, Process

def modify(ns, lproxy, dproxy):
    ns.a **= 2
    lproxy.extend(['b', 'c'])
    dproxy['b'] = 0

manager = Manager()
ns = manager.Namespace()
ns.a = 1
lproxy = manager.list()
lproxy.append('a')
dproxy = manager.dict()
dproxy['b'] = 2

p = Process(target=modify, args=(ns, lproxy, dproxy))
p.start()
p.join()

print(ns.a)       # 1
print(lproxy)     # ['a', 'b', 'c']
print(dproxy)     # {'b': 0}

Manager 支持的共享方式：

  • Namespace：可分享的命名空间
  • Value/Array：共享 ctypes 对象
  • dict/list：可分享的 dict/list，支持对应方法
  • Condition/Event/Lock/Queue/Semaphore：可分享的同步原语

分布式进程间通信

使用 Manager 和 Queue 可以实现简单的分布式通信（C/S 模式）：

Python — 服务端
from multiprocessing.managers import BaseManager

host = '127.0.0.1'
port = 9030
authkey = 'secret'
shared_list = []

class RemoteManager(BaseManager):
    pass

RemoteManager.register('get_list', callable=lambda: shared_list)
mgr = RemoteManager(address=(host, port), authkey=authkey)
server = mgr.get_server()
server.serve_forever()
Python — 客户端
from multiprocessing.managers import BaseManager

class RemoteManager(BaseManager):
    pass

RemoteManager.register('get_list')  # 注意：没有 callable 参数
mgr = RemoteManager(address=(host, port), authkey=authkey)
mgr.connect()
l = mgr.get_list()
print(l)
l.append(1)
print(mgr.get_list())

总结

核心要点

  1. GIL 问题：多线程不能利用多核，CPU 密集型任务用多进程
  2. 进程池 Pool：比手动创建进程更高效，map 方法简单易用
  3. dummy 模块：一套 API，多进程/多线程自由切换
  4. 进程间通信：Pipe（简单）、Queue（灵活）、共享内存（高效）、Manager（分布式）
  5. 同步机制：Lock、Condition、Event、RLock、Semaphore 等与 threading 类似
  6. 经验：拿不准时直接用多进程，通过 dummy 模块方便切换
