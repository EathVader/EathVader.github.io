1什么是 MRO？

MRO（Method Resolution Order） 是 Python 在多继承情况下，决定调用哪个父类方法的查找顺序。

Python
class A:
    def greet(self):
        return "Hello from A"

class B(A):
    pass

b = B()
print(b.greet())  # "Hello from A" — B 没有 greet，向上找 A

单继承很简单，但多继承就复杂了：

菱形继承：D 继承 B 和 C，B 和 C 都继承 A。D.greet() 应该调用哪个？
Python
class A:
    def greet(self):
        return "Hello from A"

class B(A):
    def greet(self):
        return "Hello from B"

class C(A):
    def greet(self):
        return "Hello from C"

class D(B, C):
    pass

d = D()
print(d.greet())  # "Hello from B" — 为什么？看 MRO
💡 查看 MRO
print(D.__mro__)
# (<class '__main__.D'>, <class '__main__.B'>, <class '__main__.C'>, <class '__main__.A'>, <class 'object'>)

print(D.mro())
# 同上

2单继承的 MRO

单继承的 MRO 很简单，就是从子类到父类再到 object 的线性顺序：

Python
class A: pass
class B(A): pass
class C(B): pass

print(C.__mro__)
# (C, B, A, object)

3多继承的 MRO

多继承时，Python 使用 C3 线性化算法（Python 2.3+）来确定 MRO。C3 算法保证：

  • 单调性：子类的 MRO 中，父类的相对顺序不变
  • 局部优先：声明时靠前的父类优先
  • 一致性扩展：MRO 与父类的 MRO 一致
Python
class A: pass
class B(A): pass
class C(A): pass
class D(B, C): pass

# C3 算法计算结果：
# D.__mro__ = (D, B, C, A, object)
# 注意：B 在 C 前面（声明顺序），A 在 B 和 C 后面（局部优先）

4C3 线性化算法详解

C3 算法的公式：

📐 C3 线性化公式
L[C(B1...BN)] = C + merge(L[B1]...L[BN], B1...BN)

其中：

  • L[C] 是类 C 的线性化（MRO）
  • L[object] = [object]
  • merge 是合并操作，从多个列表中按规则提取元素

Merge 算法规则

merge 操作从多个列表中按顺序提取元素：

  1. 取第一个列表的第一个元素（称为 head）
  2. 检查这个 head 是否出现在其他列表的尾部（tail）
  3. 如果不在任何尾部，提取它，从所有列表中删除，继续
  4. 如果在某个尾部，跳过它，检查下一个列表的 head
  5. 如果所有 head 都在某个尾部，说明 MRO 冲突，无法线性化

5Merge 算法逐步演示

以 class D(B, C) 为例，其中 B(A)，C(A)：

计算过程
# 已知：
L[object] = [object]
L[A] = A + merge(L[object], [object]) = [A, object]
L[B] = B + merge(L[A], [A]) = B + merge([A, object], [A])
     = B + [A, object] = [B, A, object]
L[C] = C + merge(L[A], [A]) = C + merge([A, object], [A])
     = C + [A, object] = [C, A, object]

# 计算 L[D]：
L[D] = D + merge(L[B], L[C], [B, C])
     = D + merge([B, A, object], [C, A, object], [B, C])

# merge 步骤：
# 1. head = B, B 不在 [C, A, object] 或 [C] 的尾部 → 取 B
#    剩余: merge([A, object], [C, A, object], [C])
# 2. head = A, A 在 [C, A, object] 的尾部 → 跳过
# 3. head = C, C 不在 [A, object] 或 [] 的尾部 → 取 C
#    剩余: merge([A, object], [A, object], [])
# 4. head = A, A 不在 [A, object] 或 [] 的尾部 → 取 A
#    剩余: merge([object], [object], [])
# 5. head = object → 取 object
#    剩余: merge([], [], [])

# 最终结果：
L[D] = [D, B, C, A, object]  ✅
Merge 过程：从三个列表中按规则提取元素，得到最终 MRO

6MRO 冲突示例

不是所有的多继承都能计算出 MRO。当父类顺序矛盾时，Python 会抛出 TypeError：

Python（MRO 冲突）
class A: pass
class B(A): pass
class C(A): pass

# 冲突！D 要求 B 在 C 前面，但 E 要求 C 在 B 前面
class D(B, C): pass  # MRO: D, B, C, A, object
class E(C, B): pass  # MRO: E, C, B, A, object

class F(D, E): pass  # ❌ TypeError!
# D 说 B 在 C 前面，E 说 C 在 B 前面，矛盾！
🚨 错误信息
TypeError: Cannot create a consistent method resolution
order (MRO) for bases B, C

解决方法：统一父类声明顺序，避免矛盾。

7经典类 vs 新式类

特性经典类（Python 2）新式类（Python 2.2+ / Python 3）
定义class A:class A(object):
Python 3不存在（所有类都是新式类）默认
MRO 算法深度优先（DFS）C3 线性化
菱形继承可能重复访问基类每个类只出现一次
super()不支持支持
Python
# Python 2 经典类（深度优先）
class A: pass
class B(A): pass
class C(A): pass
class D(B, C): pass
# 经典类 MRO: D → B → A → C → A（A 出现两次！）

# Python 3 新式类（C3 线性化）
class A: pass  # 等价于 class A(object):
class B(A): pass
class C(A): pass
class D(B, C): pass
# 新式类 MRO: D → B → C → A → object（每个类只出现一次）
✅ Python 3 中的新式类

Python 3 中所有类默认继承 object，都是新式类。经典类只在 Python 2 中存在。

8总结

要点说明
MRO 是什么方法解析顺序，决定多继承时调用哪个父类方法
C3 算法保证单调性、局部优先、一致性扩展
Merge 规则取 head，检查不在其他尾部，提取并删除
查看 MROClassName.__mro__ 或 ClassName.mro()
MRO 冲突父类声明顺序矛盾时抛出 TypeError
经典类 vs 新式类Python 3 只有新式类，使用 C3 算法
✅ 最佳实践
  • 避免过深的继承层次（不超过 3 层）
  • 多继承时保持父类声明顺序一致
  • 使用 super() 调用父类方法，而不是直接调用
  • 用 ClassName.__mro__ 调试继承问题
  • 优先使用组合而非继承
