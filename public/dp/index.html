1 为什么需要 DP / Why DP

📝

类比：做数学题时记笔记 / Analogy: Taking Notes

想象你在做一道复杂的数学题，中间需要反复计算 3 × 7 + 2。你会每次都重新算吗？当然不会——你会把结果 23 记在草稿纸上，下次直接用。DP 就是这个思路：把已经算过的结果记下来，避免重复计算。

看一个具体例子：计算斐波那契数 fib(5)，如果用朴素递归：

fib(5) 的递归树：f(3) 被计算 2 次，f(2) 被计算 3 次 —— 这就是"重叠子问题"
⚠️ 朴素递归的代价

朴素递归 fib(n) 的时间复杂度是 O(2ⁿ)，fib(50) 需要约 10¹⁵ 次计算。而用 DP 只需 O(n)，50 次计算就够了。

2 什么是 DP / What is DP

🧱

类比：搭积木 / Analogy: Building Blocks

DP 就像搭积木：你不会直接搭一座城堡，而是先搭好地基，再搭墙壁，最后搭屋顶。每一层都建立在前一层的基础上。DP 也是如此：先解决最小的子问题，逐步构建出最终答案。

动态规划的两个核心条件：

最优子结构
大问题的最优解
包含子问题的最优解
+
重叠子问题
同一个子问题
被反复求解
=
适合用 DP
记住子问题的解
避免重复计算

两种实现方式

两种方式结果相同，Bottom-Up 通常更快（无递归开销），Top-Down 更直观
💰

类比：零钱兑换 / Analogy: Coin Change

你要用硬币凑出 11 元。你不会随便试——你会想："如果我知道凑出 10 元最少要几枚硬币，那凑 11 元只需要在此基础上加一枚 1 元硬币"。这就是 DP 的状态转移：dp[11] = dp[10] + 1。

🔑 DP 解题四步法
  1. 定义状态：dp[i] 代表什么？
  2. 状态转移方程：dp[i] 和 dp[i-1]... 的关系是什么？
  3. 初始条件：最小子问题的答案是什么？
  4. 计算顺序：从小到大还是从大到小？

3 从递归到 DP / From Recursion to DP

很多人对 DP 感到困惑，是因为被"状态转移"、"状态方程"这些术语吓到了。其实 DP 就是递归的记忆化过程——把已经算过的结果记下来，避免重复计算。

第一步：先写递归

遇到 DP 问题，先别想什么状态方程，先写出最朴素的递归解法。以斐波那契为例：

朴素递归 —— 大量重复计算 ❌
int fib(int n) {
    if (n == 1 || n == 2) return 1;
    return fib(n - 1) + fib(n - 2);
}

这个函数有大量的重复计算。计算 fib(10) 时，会分别计算 fib(9) 和 fib(8)；而计算 fib(9) 时，又会计算 fib(8) 和 fib(7)……fib(8) 被算了两次！

fib(6) 的递归树：f(4) 算了 2 次，f(3) 算了 3 次 —— 这就是"重叠子问题"

测试一下 fib(50)，朴素递归已经要等很久才出结果了。时间复杂度是 O(2ⁿ)。

第二步：加记忆化（Memoization）

最简单的优化：多开一个数组，把算过的结果存起来。

记忆化递归 —— 秒算 fib(50) ✅
int[] memo = new int[100]; // 初始化为 0
int fib(int n) {
    if (n == 1 || n == 2) return 1;
    if (memo[n] != 0) return memo[n];  // 算过就直接返回
    return memo[n] = fib(n - 1) + fib(n - 2);
}

这就是所谓的 带备忘的自顶向下法（Top-Down with Memoization）。思路是：数组里的每一个数，都是依靠前面两个数的和计算得出——这就是最优子结构。但如果不记忆化，就会有大量重复计算——这就是重叠子问题。

每个值只计算一次，时间复杂度从 O(2ⁿ) 降到 O(n)

第三步：改写成循环（Bottom-Up）

记忆化递归虽然快，但递归有函数调用开销，且某些语言（如 Java、Python）不支持尾递归，容易栈溢出。改成循环既节省递归切换的时间，又避免了栈溢出。

Bottom-Up DP —— 推荐写法 ✅
int fib(int n) {
    int a = 0, b = 1, c;
    for (int i = 1; i < n; i++) {
        c = a + b;
        a = b;
        b = c;
    }
    return b;
}

这就是所谓的 自底向上法（Bottom-Up / Tabulation）。不需要递归，速度比上面两种都快。而且因为 dp[i] 只依赖前两个值，所以连数组都不需要，只用两个变量即可——空间复杂度 O(1)。

💡 三种写法对比
写法时间空间特点
朴素递归O(2ⁿ) ❌O(n)大量重复计算
记忆化递归O(n) ✅O(n)自顶向下，直观
循环 DPO(n) ✅O(1) ✅自底向上，最快

4 斐波那契数列 / Fibonacci

最经典的 DP 入门题：fib(n) = fib(n-1) + fib(n-2)。

🔑 DP 四步法
  1. 状态：dp[i] = 第 i 个斐波那契数
  2. 转移：dp[i] = dp[i-1] + dp[i-2]
  3. 初始：dp[0] = 0, dp[1] = 1
  4. 顺序：从小到大（i = 2, 3, ..., n）
每个格子 = 前两个格子之和，从左到右填充

三种实现对比

朴素递归 O(2ⁿ) ❌
int fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2); // 大量重复计算
}
记忆化递归 O(n) ✅
int[] memo = new int[n + 1];
Arrays.fill(memo, -1);
int fib(int n) {
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n]; // 查备忘录
    return memo[n] = fib(n - 1) + fib(n - 2);
}
Bottom-Up DP O(n) ✅（推荐）
int fib(int n) {
    if (n <= 1) return n;
    int prev2 = 0, prev1 = 1;
    for (int i = 2; i <= n; i++) {
        int cur = prev1 + prev2;
        prev2 = prev1;
        prev1 = cur;
    }
    return prev1; // 空间优化到 O(1)
}
💡 空间优化

因为 dp[i] 只依赖 dp[i-1] 和 dp[i-2]，所以不需要整个数组，只需两个变量即可。这是 DP 中常见的滚动变量优化。

实战：打家劫舍 / House Robber

给你一个数组，挑一串数字使和最大化，但每两个数不能相邻。比如 [2, 7, 9, 3, 1]，最优解是选 2 + 9 + 1 = 12。

🏠

类比：小偷的选择 / Analogy: The Robber's Choice

你是一条街上的小偷，每间房有一定金额的现金。但相邻的两间房有警报系统，如果同时偷了相邻的两间就会触发警报。对于每间房，你有两种选择：偷或不偷。选了这间就不能选下一间。

绿色 = 偷的房子，灰色 = 不偷的房子。不能选相邻的房子。

思路是数组里的每一个数，我有两种选择：选或者不选。选了这个就不能选下一个。递归地求解，然后进行比较，选取较大的那个，并且用数组记忆结果。

House Robber —— 记忆化递归（自顶向下）
class Solution {
    private int[] memo;
    public int rob(int[] nums) {
        memo = new int[nums.length];
        Arrays.fill(memo, -1);
        return rob(nums, 0);
    }
    private int rob(int[] nums, int i) {
        if (i >= nums.length) return 0;
        if (memo[i] != -1) return memo[i];    // 查备忘录
        // 选或不选，取较大值
        int max = Math.max(rob(nums, i + 1),    // 不偷这间
                           nums[i] + rob(nums, i + 2)); // 偷这间
        return memo[i] = max;
    }
}
House Robber —— 循环 DP（自底向上，推荐）
int rob(int[] nums) {
    if (nums.length == 0) return 0;
    if (nums.length == 1) return nums[0];
    int[] dp = new int[nums.length];
    dp[0] = nums[0];
    dp[1] = Math.max(nums[0], nums[1]);
    for (int i = 2; i < nums.length; i++) {
        // 核心公式：max(不偷这间, 偷这间)
        dp[i] = Math.max(dp[i - 1], nums[i] + dp[i - 2]);
    }
    return dp[nums.length - 1];
}

这种方法就是所谓的自底向上法，不需要递归切换，速度比记忆化递归更快。核心公式 dp[i] = max(dp[i-1], nums[i] + dp[i-2]) 就是状态转移方程。

🔑 从递归到 DP 的通用套路
  1. 先写递归：用递归描述问题，理清"选或不选"的决策逻辑
  2. 加备忘录：用数组记忆已经算过的结果（自顶向下）
  3. 改写成循环：从最小子问题出发，逐步填表到目标（自底向上）
  4. 空间优化：如果 dp[i] 只依赖前面几个值，可以用滚动变量代替数组

这个套路适用于几乎所有 DP 问题！

5 零钱兑换 / Coin Change

给定硬币面额 [1, 2, 5]，凑出金额 11 最少需要几枚硬币？

🔑 DP 四步法
  1. 状态：dp[i] = 凑出金额 i 所需的最少硬币数
  2. 转移：dp[i] = min(dp[i - coin] + 1)，对每种硬币
  3. 初始：dp[0] = 0（凑出 0 元需要 0 枚硬币）
  4. 顺序：从 1 到 amount
dp[5]=1（一枚 5 元硬币），dp[11]=3（5+5+1）
Coin Change - Java
int coinChange(int[] coins, int amount) {
    int[] dp = new int[amount + 1];
    Arrays.fill(dp, amount + 1); // 初始化为不可能的大值
    dp[0] = 0;
    for (int i = 1; i <= amount; i++) {
        for (int coin : coins) {
            if (coin <= i)
                dp[i] = Math.min(dp[i], dp[i - coin] + 1);
        }
    }
    return dp[amount] > amount ? -1 : dp[amount];
}

5 零钱兑换 / Coin Change

有 n 个物品，每个有重量 w[i] 和价值 v[i]。背包容量为 W，每个物品只能选或不选，求最大价值。

🎒

类比：旅行打包 / Analogy: Packing for a Trip

你的行李箱只能装 10kg。面前有笔记本电脑（2kg, 价值 6）、相机（3kg, 价值 5）、书（4kg, 价值 4）、衣服（5kg, 价值 7）。你需要选择总重量不超过 10kg 的组合，使总价值最大。

🔑 DP 四步法
  1. 状态：dp[i][w] = 前 i 个物品、容量为 w 时的最大价值
  2. 转移：dp[i][w] = max(dp[i-1][w], dp[i-1][w-w[i]] + v[i])
  3. 初始：dp[0][*] = 0（没有物品时价值为 0）
  4. 顺序：i 从 1 到 n，w 从 0 到 W
决策树：每个物品选或不选，DP 用表格避免重复探索
0/1 Knapsack - Java
int knapsack(int[] w, int[] v, int W) {
    int n = w.length;
    int[] dp = new int[W + 1]; // 空间优化：一维数组
    for (int i = 0; i < n; i++) {
        for (int j = W; j >= w[i]; j--) { // 逆序遍历！
            dp[j] = Math.max(dp[j], dp[j - w[i]] + v[i]);
        }
    }
    return dp[W];
}
⚠️ 为什么逆序遍历？

0/1 背包中每个物品只能选一次。如果正序遍历，dp[j - w[i]] 可能已经包含了物品 i，导致物品被重复选取。逆序遍历保证每个物品只被考虑一次。

7 最长公共子序列 / LCS

给定两个字符串，找出它们的最长公共子序列（不要求连续）。例如 "ABCBDAB" 和 "BDCAB" 的 LCS 是 "BCAB"（长度 4）。

🔑 DP 四步法
  1. 状态：dp[i][j] = A 的前 i 个字符与 B 的前 j 个字符的 LCS 长度
  2. 转移：若 A[i]==B[j]，dp[i][j] = dp[i-1][j-1] + 1；否则 dp[i][j] = max(dp[i-1][j], dp[i][j-1])
  3. 初始：dp[0][*] = dp[*][0] = 0
  4. 顺序：i 从 1 到 m，j 从 1 到 n
绿色格子 = 字符匹配（来自左上角 +1），其余 = max(上, 左)
LCS - Java
int lcs(String a, String b) {
    int m = a.length(), n = b.length();
    int[][] dp = new int[m + 1][n + 1];
    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (a.charAt(i - 1) == b.charAt(j - 1))
                dp[i][j] = dp[i - 1][j - 1] + 1;
            else
                dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);
        }
    }
    return dp[m][n];
}

8 最长回文子串 / Longest Palindromic Substring

给定字符串 "babad"，找出最长的回文子串。答案是 "bab" 或 "aba"。

🔑 DP 四步法
  1. 状态：dp[i][j] = s[i..j] 是否为回文
  2. 转移：dp[i][j] = (s[i]==s[j]) && dp[i+1][j-1]
  3. 初始：dp[i][i] = true（单字符），dp[i][i+1] = (s[i]==s[i+1])
  4. 顺序：按子串长度从小到大（len = 1, 2, ..., n）
两种方法都是 O(n²)，中心扩展法空间 O(1)，DP 法空间 O(n²)
Longest Palindromic Substring - Java
String longestPalindrome(String s) {
    int n = s.length(), start = 0, maxLen = 1;
    boolean[][] dp = new boolean[n][n];
    // 单字符
    for (int i = 0; i < n; i++) dp[i][i] = true;
    // 按长度填表
    for (int len = 2; len <= n; len++) {
        for (int i = 0; i <= n - len; i++) {
            int j = i + len - 1;
            if (s.charAt(i) == s.charAt(j)) {
                dp[i][j] = (len == 2) || dp[i + 1][j - 1];
                if (dp[i][j] && len > maxLen) {
                    start = i; maxLen = len;
                }
            }
        }
    }
    return s.substring(start, start + maxLen);
}
💡 更优解法：Manacher 算法

Manacher 算法可以在 O(n) 时间内找到最长回文子串，但理解难度较大。面试中 DP 或中心扩展法已经足够。

9 DP 模式总结 / DP Patterns

大多数 DP 问题可以归类为以下几种模式：

模式典型问题状态定义关键特征
线性 DP斐波那契、爬楼梯、打家劫舍dp[i] = 到第 i 个的最优解一维数组，从左到右
背包 DP0/1 背包、完全背包、分割等和dp[i][w] = 前 i 个物品容量 w选/不选决策
区间 DP回文子串、矩阵链乘、戳气球dp[i][j] = 区间 [i,j] 的最优解按区间长度枚举
双串 DPLCS、编辑距离、正则匹配dp[i][j] = A 前 i 个与 B 前 j 个二维表格
树形 DP二叉树最大路径和、打家劫舍 IIIdp[node] = 以 node 为根的最优解后序遍历
状压 DPTSP、任务分配dp[mask] = 已访问集合为 mask位运算表示状态
DP 解题五步：识别子问题 → 定义状态 → 转移方程 → 初始值 → 填表
