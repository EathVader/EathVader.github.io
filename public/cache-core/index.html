1 存储层次与缓存基础

计算机存储层次

存储层级访问延迟典型容量
寄存器1 周期极少
L1/L2 Cache几周期KB-MB
L3 Cache十几周期MB
内存几十-几百周期GB
磁盘/SSD毫秒级TB

局部性原理

⏰ 时间局部性
访问过的数据很可能再次被访问
📍 空间局部性
附近地址很可能也被访问

Cache Line 映射方式

方式说明
直接映射每个地址只能映射到唯一位置
组相联每组有多个 Cache Line（如 8-way）
全相联任意地址可映射到任意位置
📏 Cache Line 大小

典型大小 64 字节，是 Cache 与内存同步的最小单位。

2 CPU 缓存一致性

多核缓存架构

L1/L2
每核独有
L3
所有核心共享

MESI 协议

状态含义
Modified数据已修改，未同步到内存
Exclusive独占，无副本
Shared共享，多核持有相同数据
Invalid无效，需从内存加载

内存屏障（Memory Fence）

屏障类型作用
Store Barrier强制刷新写缓冲到内存
Load Barrier强制使缓存失效，重新读取
Full Barrier兼具两者
⚠️ False Sharing（伪共享）

不同线程访问同一 Cache Line 上的不同数据，导致缓存失效。解决方案：字节填充（Padding）。

4 缓存问题与解决方案

缓存穿透

❌ 问题

查询不存在的数据，每次都打到后端数据库。

解决方案：布隆过滤器 · 缓存空值 · 请求合法性校验

缓存击穿

❌ 问题

热点 key 过期瞬间，大量请求同时打到后端。

解决方案：互斥锁 · 永不过期（逻辑过期） · 提前更新

缓存雪崩

❌ 问题

大量 key 同时过期，或 Cache 服务故障。

解决方案：随机过期时间 · 多级缓存 · 熔断/限流 · 预热

服务过载预防

Client 端
限制并发、请求排队
Server 端
熔断、降级、限流
Cache 端
预热、渐进恢复

5 分布式缓存

缓存集群模式

模式说明
主从主节点写，从节点读
分片哈希/一致性哈希分配到不同节点
Redis Cluster自动分片 + 哨兵
✅ 性能优化建议
  • 优先使用本地缓存（Guava Cache、Caffeine）
  • 热点数据用分布式缓存（Redis）
  • 合理设置过期时间，避免同时过期
  • 监控命中率，低于 80% 需优化
  • 避免大对象，超过 Cache Line 会降低效率
  • 批量操作，减少网络往返
