3 冒泡排序 / Bubble Sort

🫧

类比 / Analogy

像水中的气泡一样，大的元素不断"浮"到数组末尾。每一轮把当前最大的元素冒泡到正确位置。

相邻元素比较
→
逆序则交换
→
一轮后最大值
到达末尾
→
重复 n-1 轮
void bubbleSort(int[] a) {
    for (int i = 0; i < a.length - 1; i++) {
        boolean swapped = false;
        for (int j = 0; j < a.length - 1 - i; j++) {
            if (a[j] > a[j + 1]) {
                swap(a, j, j + 1);
                swapped = true;
            }
        }
        if (!swapped) break; // 优化：无交换说明已有序
    }
}

4 选择排序 / Selection Sort

在未排序区
找最小值
→
与未排序区
第一个交换
→
有序区 +1
重复 n-1 轮
void selectionSort(int[] a) {
    for (int i = 0; i < a.length - 1; i++) {
        int min = i;
        for (int j = i + 1; j < a.length; j++)
            if (a[j] < a[min]) min = j;
        swap(a, i, min);
    }
}
⚠️ 不稳定

选择排序是不稳定的。例如 [5a, 5b, 2]，第一轮把 2 和 5a 交换，5a 跑到了 5b 后面。

5 插入排序 / Insertion Sort

🃏

类比 / Analogy

像打扑克牌整理手牌：每摸一张新牌，从右往左找到合适位置插入。

void insertionSort(int[] a) {
    for (int i = 1; i < a.length; i++) {
        int key = a[i], j = i - 1;
        while (j >= 0 && a[j] > key) {
            a[j + 1] = a[j]; j--;
        }
        a[j + 1] = key;
    }
}
💡 插入排序的优势

对近乎有序的数据，插入排序接近 O(n)。这就是为什么 TimSort（Python/Java 的默认排序）在小数组或部分有序时切换到插入排序。

6 希尔排序 / Shell Sort

插入排序的改进版：先用大步长分组排序，逐步缩小步长，最后步长为 1 时等价于插入排序，但此时数组已近乎有序。

gap = n/2
→
对每组做
插入排序
→
gap /= 2
→
gap=1 时
最终排序
void shellSort(int[] a) {
    for (int gap = a.length / 2; gap > 0; gap /= 2) {
        for (int i = gap; i < a.length; i++) {
            int key = a[i], j = i;
            while (j >= gap && a[j - gap] > key) {
                a[j] = a[j - gap]; j -= gap;
            }
            a[j] = key;
        }
    }
}

7 归并排序 / Merge Sort

🪓

分治思想

把数组一分为二，分别排序，再合并两个有序数组。递归到底就是单个元素（天然有序）。

void mergeSort(int[] a, int lo, int hi) {
    if (lo >= hi) return;
    int mid = (lo + hi) / 2;
    mergeSort(a, lo, mid);
    mergeSort(a, mid + 1, hi);
    merge(a, lo, mid, hi);
}

void merge(int[] a, int lo, int mid, int hi) {
    int[] tmp = Arrays.copyOfRange(a, lo, hi + 1);
    int i = 0, j = mid - lo + 1, k = lo;
    while (i <= mid - lo && j <= hi - lo) {
        a[k++] = tmp[i] <= tmp[j] ? tmp[i++] : tmp[j++];
    }
    while (i <= mid - lo) a[k++] = tmp[i++];
    while (j <= hi - lo) a[k++] = tmp[j++];
}

8 快速排序 / Quick Sort

🔑 核心思想：分区 (Partition)

选一个基准元素（pivot），把数组分成两部分：左边 ≤ pivot，右边 ≥ pivot。然后递归排序左右两部分。

void quickSort(int[] a, int lo, int hi) {
    if (lo >= hi) return;
    int pivotIdx = partition(a, lo, hi);
    quickSort(a, lo, pivotIdx - 1);
    quickSort(a, pivotIdx + 1, hi);
}

int partition(int[] a, int lo, int hi) {
    int pivot = a[hi], i = lo; // 选最后一个为 pivot
    for (int j = lo; j < hi; j++) {
        if (a[j] < pivot) swap(a, i++, j);
    }
    swap(a, i, hi);
    return i;
}
⚠️ 最坏情况 O(n²)

当数组已有序且每次选第一个/最后一个为 pivot 时退化为 O(n²)。解决方案：随机选 pivot 或 三数取中。

9 堆排序 / Heap Sort

📐 堆的性质

大根堆：每个节点 ≥ 其子节点。用数组存储：节点 i 的子节点为 2i+1 和 2i+2。

建堆
O(n) 自底向上
→
堆顶（最大值）
与末尾交换
→
堆大小 -1
下沉调整
→
重复 n-1 次
void heapSort(int[] a) {
    int n = a.length;
    // 建堆：从最后一个非叶节点开始下沉
    for (int i = n / 2 - 1; i >= 0; i--) siftDown(a, i, n);
    // 排序：每次取堆顶放到末尾
    for (int i = n - 1; i > 0; i--) {
        swap(a, 0, i);
        siftDown(a, 0, i);
    }
}

void siftDown(int[] a, int i, int size) {
    while (2 * i + 1 < size) {
        int child = 2 * i + 1;
        if (child + 1 < size && a[child + 1] > a[child]) child++;
        if (a[i] >= a[child]) break;
        swap(a, i, child);
        i = child;
    }
}
💡 堆排序的优势

最坏情况也是 O(n log n)，且只需 O(1) 额外空间。缺点是缓存不友好（跳跃访问），实践中比快排慢。

10 计数 · 桶 · 基数 / Non-comparison Sorts

比较排序的下界是 O(n log n)。要突破这个限制，需要利用数据的特殊性质。

计数排序 / Counting Sort

适用于整数且范围 k 不大的情况。统计每个值出现的次数，然后按顺序输出。

void countingSort(int[] a, int maxVal) {
    int[] count = new int[maxVal + 1];
    for (int v : a) count[v]++;
    int idx = 0;
    for (int v = 0; v <= maxVal; v++)
        while (count[v]-- > 0) a[idx++] = v;
}

基数排序 / Radix Sort

从最低位到最高位，每一位用计数排序。适合整数或等长字符串。

void radixSort(int[] a) {
    int max = Arrays.stream(a).max().orElse(0);
    for (int exp = 1; max / exp > 0; exp *= 10)
        countingSortByDigit(a, exp);
}

桶排序 / Bucket Sort

将数据分到若干个桶中，每个桶内部排序，最后合并。适合均匀分布的浮点数。

🎯 非比较排序的适用条件

计数排序：整数，范围 k = O(n)
基数排序：整数/等长字符串，位数 d 较小
桶排序：数据均匀分布在某个范围内
