1 问题背景：为什么需要文档去重？

互联网上充斥着大量近似重复的内容——镜像网站、内容复制、嵌入广告、少量修改。对搜索引擎来说，收录重复网页只会浪费存储和计算资源。

🎯 核心挑战
Google 收录了数十亿网页，每天新增数百万。如果每条新数据都和库中每条记录计算余弦相似度，计算量是天文数字。
我们需要一种方法：把文档压缩成"指纹"，通过比较指纹快速判断相似度。

传统 hash（如 MD5）的问题：输入哪怕只变一个字节，hash 就完全不同。我们需要的是局部敏感哈希（LSH）——相似的输入产生相似的 hash。

💡 两种思路
• SimHash（指纹识别文档）：把文档压缩成一个 64-bit 指纹，通过海明距离判断相似度
• MinHash（集合相似度）：通过随机哈希估算两个集合的 Jaccard 相似系数

2 SimHash：指纹识别文档

SimHash 由 Moses Charikar 提出，Google 用它实现网页去重。核心思想：把高维特征向量降维成一个 f-bit 的指纹，相似文档的指纹也相似。

2.1 算法流程

整个过程分为 5 步：分词 → Hash → 加权 → 合并 → 降维

Step 1: 特征提取（分词 + 权重）

# Extract features with weights (e.g., TF-IDF)
doc = "the cat sat on the mat"
features = {"the": 2, "cat": 1, "sat": 1, "on": 1, "mat": 1}
# Or use n-gram: {"th": 1, "he": 1, "e ": 1, " c": 1, ...}

Step 2-3: Hash 并加权

# For each feature, compute hash and apply weight
# hash("cat") = 100110  (example 6-bit)
# weight = 1
# bit=1 → +weight, bit=0 → -weight
# result: [+1, -1, -1, +1, +1, -1]

Step 4-5: 合并向量 → 生成指纹

# Sum all weighted hash vectors
# V = [+13, +108, -22, -5, -32, +55]
# Positive → 1, Negative → 0
# SimHash = 110001

2.2 Python 实现

class SimHash:
    def __init__(self, tokens, hashbits=64):
        self.hashbits = hashbits
        self.hash = self._simhash(tokens)

    def _simhash(self, tokens):
        v = [0] * self.hashbits
        for t in tokens:
            h = self._string_hash(t)
            for i in range(self.hashbits):
                bitmask = 1 << i
                if h & bitmask:
                    v[i] += 1   # bit=1 → +1
                else:
                    v[i] -= 1   # bit=0 → -1
        fingerprint = 0
        for i in range(self.hashbits):
            if v[i] >= 0:
                fingerprint |= (1 << i)
        return fingerprint

    def _string_hash(self, source):
        if not source:
            return 0
        x = ord(source[0]) << 7
        m, mask = 1000003, (1 << self.hashbits) - 1
        for c in source:
            x = ((x * m) ^ ord(c)) & mask
        return x ^ len(source)

    def hamming_distance(self, other):
        x = (self.hash ^ other.hash) & ((1 << self.hashbits) - 1)
        d = 0
        while x:
            d += 1
            x &= x - 1
        return d
🧩 SimHash 的直觉：指纹识别文档
就像人的指纹——每个人的指纹独一无二，但双胞胎的指纹会非常相似。
SimHash 为每篇文档生成一个"数字指纹"，相似文档的指纹只有少数几位不同。

关键性质：
• 它是一个 hash 函数（能压缩任意长度文档为固定位数）
• 相似文档的 hash 值也相似（局部敏感性）

3 海明距离与相似度判定

两个二进制串的海明距离 = 不同位的数量。SimHash 通过海明距离判断文档相似度。

# Hamming distance via XOR
A = 0b110001
B = 0b110100
xor = A ^ B  # 0b000101 → two 1-bits → distance = 2

def hamming_distance(a, b):
    x = a ^ b
    d = 0
    while x:
        d += 1
        x &= x - 1   # clear lowest set bit (Brian Kernighan's trick)
    return d
💡 经验阈值
Google 的实践：64-bit SimHash，海明距离 ≤ 3 即认为是近似重复文档。
具体阈值需要根据业务场景和数据集调整。
海明距离含义示例
0完全相同同一篇文章
1-3近似重复少量修改、嵌入广告
4-10部分相似同一话题不同文章
>10不相似完全不同的内容

方法思路问题

4.2 分块索引（抽屉原理）

🔑 核心思想：抽屉原理
将 64-bit 指纹分成 4 块（每块 16 位）。如果两个指纹的海明距离 ≤ 3，那么至少有一块是完全相同的。

因为 3 个不同的 bit 最多分布在 3 个块中，第 4 个块必然完全匹配。 
# Split 64-bit fingerprint into 4 blocks of 16 bits
# Build 4 sorted tables, each with a different block as prefix
# For query F:
#   1. In each table, binary search for matching 16-bit prefix
#   2. For candidates, check full 64-bit hamming distance ≤ 3

# With 2^34 (~8 billion) fingerprints:
# Each table returns ~2^(34-16) = 262144 candidates
# Much cheaper than 8 billion full comparisons!
📊 时空权衡
表数量每表前缀位数候选数量存储倍数
4162^(d-16)4×
16282^(d-28)16×
表越多 → 候选越少（更快），但存储越大。这就是时间与空间的权衡。

5 MinHash：集合相似度

MinHash 是另一种 LSH 方法，由 Andrei Broder 提出，用于快速估算两个集合的 Jaccard 相似系数。

5.1 Jaccard 系数

# Jaccard similarity = |A ∩ B| / |A ∪ B|
A = {"the", "cat", "sat", "on", "mat"}
B = {"the", "cat", "sat", "on", "a", "mat"}
# A ∩ B = {"the", "cat", "sat", "on", "mat"} → 5
# A ∪ B = {"the", "cat", "sat", "on", "mat", "a"} → 6
# J(A, B) = 5/6 ≈ 0.833

5.2 MinHash 原理

🧩 MinHash 的直觉：集合相似度
想象把两个集合的所有元素放进一个袋子里，随机抽一个。
如果两个集合越相似，随机抽到的"最小元素"相同的概率就越大。

数学上：Pr[h_min(A) = h_min(B)] = J(A, B)
即两个集合经过 hash 后最小值相同的概率 = Jaccard 系数。

5.3 两种实现方式

方法 1：多个 Hash 函数

# Use K hash functions, get K min-hash values per set
# Similarity ≈ number of matching min-values / K
import hashlib

def minhash_multi(tokens, num_hashes=100):
    """Generate MinHash signature using multiple hash functions."""
    signature = []
    for i in range(num_hashes):
        min_val = float('inf')
        for token in tokens:
            h = hash(f"{i}_{token}") & 0xFFFFFFFF
            min_val = min(min_val, h)
        signature.append(min_val)
    return signature

def minhash_similarity(sig_a, sig_b):
    """Estimate Jaccard similarity from MinHash signatures."""
    matches = sum(1 for a, b in zip(sig_a, sig_b) if a == b)
    return matches / len(sig_a)

方法 2：单个 Hash 函数取 K 个最小值

# Hash all elements once, keep K smallest values
def minhash_single(tokens, k=100):
    """Generate MinHash signature using single hash, K smallest."""
    hashes = sorted(hash(t) & 0xFFFFFFFF for t in tokens)
    return hashes[:k]

# Similarity = |min_k(A) ∩ min_k(B)| / |min_k(A) ∪ min_k(B)|

5.4 MinHash 矩阵降维

MinHash 的真正威力在于海量文档的场景。假设有 n 个文档，每个文档的词表维度为 m：

原始矩阵MinHash 签名矩阵
维度n × mn × k
示例100万 × 50万100万 × 200
比较代价O(m) per pairO(k) per pair
💡 为什么 MinHash 能降维？
每个排列（hash 函数）把一篇文档映射成一个整数。
K 个 hash 函数 → K 个整数 → 一个 K 维的"签名"。
当 K << m 时，计算量大幅下降，而相似度估算仍然准确。

🔑 如何选择？
• 需要极致存储效率和海量数据快速去重 → SimHash
• 需要更精确的相似度估算和灵活的阈值调整 → MinHash
• 实际系统中，两者常常结合使用：SimHash 做粗筛，MinHash 做精排
