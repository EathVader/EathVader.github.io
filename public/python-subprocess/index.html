Python subprocess PIPE 使用指南

原创 · 2026-04-02 · 阅读约 12 分钟

为什么需要 subprocess？

在 Python 中调用外部命令是常见需求：执行 shell 脚本、调用系统工具、启动子进程等。早期的 os.system() 和 os.popen() 功能有限，Python 2.4 引入了 subprocess 模块，提供了更强大、更安全的子进程管理能力。

⚠️ 注意：从 Python 3.5 开始，官方推荐使用 subprocess.run() 作为高级接口，subprocess.Popen 作为底层接口。本文以 Popen 为主，因为它能更好地展示 PIPE 的工作机制。

subprocess.Popen 基础

Popen 是 subprocess 的核心类，负责创建和管理子进程：

基本用法
import subprocess

# 最简单的用法：执行命令并等待完成
proc = subprocess.Popen(['ls', '-l', '/tmp'])
proc.wait()  # 等待子进程结束

# 获取返回值
print(f"退出码: {proc.returncode}")

关键参数

参数说明默认值
args命令及参数（列表或字符串）必填

stdout标准输出None（继承父进程）

PIPE 的三种使用场景

场景一：PIPE 传递给另一个命令（管道链）

模拟 shell 中的管道操作 cmd1 | cmd2 | cmd3：

grep 示例：ps aux | grep python | wc -l
import subprocess

# 第一个进程：ps aux
p1 = subprocess.Popen(['ps', 'aux'], stdout=subprocess.PIPE)

# 第二个进程：grep python（从 p1 的 stdout 读取）
p2 = subprocess.Popen(['grep', 'python'], stdin=p1.stdout, stdout=subprocess.PIPE)

# 关闭 p1 的 stdout，让 p1 知道没有读者了
p1.stdout.close()

# 第三个进程：wc -l（从 p2 的 stdout 读取）
p3 = subprocess.Popen(['wc', '-l'], stdin=p2.stdout, stdout=subprocess.PIPE)
p2.stdout.close()

# 获取最终结果
output, _ = p3.communicate()
print(f"Python 进程数: {output.decode().strip()}")
💡 关键点：中间的进程（p1、p2）需要关闭其 stdout，否则可能导致子进程无法正常退出（因为管道仍有读者）。

场景二：在 Python 中捕获输出

将子进程的输出读取到 Python 变量中处理：

读取命令输出
import subprocess

# 方式一：communicate()（推荐）
proc = subprocess.Popen(['df', '-h'], stdout=subprocess.PIPE, stderr=subprocess.PIPE)
stdout, stderr = proc.communicate()
print(stdout.decode('utf-8'))

# 方式二：逐行读取（适合大量输出）
proc = subprocess.Popen(['tail', '-f', '/var/log/syslog'],
                       stdout=subprocess.PIPE,
                       stderr=subprocess.PIPE,
                       text=True)  # text=True 自动解码

for line in proc.stdout:
    print(line, end='')
    if 'ERROR' in line:
        print(f"发现错误: {line}")
        break

proc.terminate()  # 结束 tail 进程

场景三：向子进程输入数据

通过 stdin 向子进程发送数据：

向子进程发送输入
import subprocess

# 方式一：communicate() 发送输入
proc = subprocess.Popen(['grep', '-c', 'error'],
                       stdin=subprocess.PIPE,
                       stdout=subprocess.PIPE,
                       text=True)

log_data = """2024-01-01 INFO: started
2024-01-01 ERROR: connection failed
2024-01-01 INFO: retrying
2024-01-01 ERROR: timeout
2024-01-01 INFO: done"""

stdout, _ = proc.communicate(input=log_data)
print(f"错误行数: {stdout.strip()}")  # 输出: 2

# 方式二：写入 stdin（适合交互式命令）
proc = subprocess.Popen(['cat'],
                       stdin=subprocess.PIPE,
                       stdout=subprocess.PIPE,
                       text=True)

proc.stdin.write("Hello\n")
proc.stdin.flush()  # 必须 flush 才能立即发送
result = proc.stdout.readline()
print(f"echo: {result.strip()}")

proc.stdin.close()
proc.wait()

常见陷阱和最佳实践

陷阱一：死锁 — PIPE 缓冲区满

⚠️ 最经典的死锁场景：当子进程输出大量数据时，如果 PIPE 缓冲区（通常 64KB）满了，子进程会阻塞等待父进程读取。如果父进程在等待子进程结束（wait()），就会形成死锁。
❌ 错误写法：可能死锁
import subprocess

# 危险！如果子进程输出超过 PIPE 缓冲区大小，会死锁
proc = subprocess.Popen(['large_output_command'],
                       stdout=subprocess.PIPE,
                       stderr=subprocess.PIPE)
proc.wait()  # 可能永远阻塞！
stdout = proc.stdout.read()  # 永远不会执行到这里
✅ 正确写法：使用 communicate()
import subprocess

# communicate() 会同时读取 stdout 和 stderr，避免死锁
proc = subprocess.Popen(['large_output_command'],
                       stdout=subprocess.PIPE,
                       stderr=subprocess.PIPE)
stdout, stderr = proc.communicate()  # 安全！

陷阱二：stderr 丢失

❌ stderr 未捕获，错误信息丢失
proc = subprocess.Popen(['invalid_command'],
                       stdout=subprocess.PIPE)
# stderr 默认输出到终端，无法在 Python 中获取
✅ 正确写法：捕获 stderr
proc = subprocess.Popen(['invalid_command'],
                       stdout=subprocess.PIPE,
                       stderr=subprocess.PIPE)
stdout, stderr = proc.communicate()

if proc.returncode != 0:
    print(f"命令失败: {stderr.decode()}")

陷阱三：shell=True 的安全风险

⚠️ 命令注入风险：使用 shell=True 时，如果参数来自用户输入，可能导致命令注入攻击。
❌ 危险：shell=True + 用户输入
import subprocess

filename = input("输入文件名: ")
# 如果用户输入: file.txt; rm -rf /
subprocess.Popen(f"cat {filename}", shell=True)
# 会执行: cat file.txt; rm -rf /
✅ 安全：使用列表参数
import subprocess

filename = input("输入文件名: ")
# 即使输入包含特殊字符，也会被当作普通参数
subprocess.Popen(['cat', filename])

陷阱四：超时处理

✅ 设置超时避免无限等待
import subprocess

try:
    proc = subprocess.Popen(['slow_command'],
                           stdout=subprocess.PIPE,
                           stderr=subprocess.PIPE)
    stdout, stderr = proc.communicate(timeout=30)
except subprocess.TimeoutExpired:
    proc.kill()
    stdout, stderr = proc.communicate()
    print("命令超时，已强制终止")

subprocess.run vs Popen 对比

特性subprocess.run()subprocess.Popen
Python 版本3.5+2.4+
阻塞/非阻塞阻塞（等待完成）非阻塞（可异步控制）
PIPE 链不支持支持（管道链）
交互式输入有限支持完全支持
逐行读取不支持支持
超时控制内置 timeout 参数需手动实现
适用场景简单命令执行复杂进程管理
💡 选择建议：如果只是执行一个命令并获取结果，用 subprocess.run()；如果需要管道链、交互式输入、逐行读取等高级功能，用 subprocess.Popen。

实用代码模板

模板一：执行命令并检查返回值

run_and_check.py
import subprocess

def run_command(cmd, check=True):
    """执行命令并返回结果"""
    result = subprocess.run(
        cmd,
        stdout=subprocess.PIPE,
        stderr=subprocess.PIPE,
        text=True,
        timeout=60
    )

    if check and result.returncode != 0:
        raise RuntimeError(
            f"命令失败 (退出码 {result.returncode}):\n"
            f"stderr: {result.stderr}"
        )

    return result.stdout

# 使用
output = run_command(['git', 'status'])
print(output)

模板二：实时读取输出

stream_output.py
import subprocess
import sys

def stream_command(cmd):
    """实时流式读取命令输出"""
    proc = subprocess.Popen(
        cmd,
        stdout=subprocess.PIPE,
        stderr=subprocess.STDOUT,  # 合并 stderr 到 stdout
        text=True,
        bufsize=1  # 行缓冲
    )

    for line in proc.stdout:
        print(line, end='')
        sys.stdout.flush()  # 确保立即输出

    proc.wait()
    return proc.returncode

# 使用
exit_code = stream_command(['docker', 'build', '-t', 'myapp', '.'])

模板三：管道链封装

pipe_chain.py
import subprocess

def pipe_chain(*commands):
    """执行命令管道链 cmd1 | cmd2 | cmd3"""
    if not commands:
        raise ValueError("至少需要一个命令")

    # 启动第一个命令
    proc = subprocess.Popen(
        commands[0],
        stdout=subprocess.PIPE,
        stderr=subprocess.PIPE
    )

    # 连接后续命令
    for cmd in commands[1:]:
        proc = subprocess.Popen(
            cmd,
            stdin=proc.stdout,
            stdout=subprocess.PIPE,
            stderr=subprocess.PIPE
        )
        proc.stdout.close()

    # 获取最终结果
    stdout, stderr = proc.communicate()
    return stdout, stderr, proc.returncode

# 使用: ps aux | grep python | wc -l
stdout, stderr, code = pipe_chain(
    ['ps', 'aux'],
    ['grep', 'python'],
    ['wc', '-l']
)
print(f"结果: {stdout.decode().strip()}")

总结

  • PIPE 三种场景：管道链传递、Python 中捕获输出、向子进程输入数据
  • 避免死锁：使用 communicate() 而非 wait() + read()
  • 捕获 stderr：始终设置 stderr=subprocess.PIPE 或 stderr=subprocess.STDOUT
  • 安全第一：避免 shell=True，使用列表参数
  • 超时控制：使用 timeout 参数或手动 kill()
  • 简单场景用 run()，复杂场景用 Popen
