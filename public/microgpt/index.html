2.2 字符级分词器 / Character-Level Tokenizer

分词器的任务是把文本转换为模型能处理的数字序列。MicroGPT 使用最简单的方式：每个字符就是一个 token。

chars = ['<BOS>'] + sorted(set(''.join(docs)))
# chars = ['<BOS>', 'a', 'b', 'c', ..., 'z']  共 27 个

stoi = { ch:i for i, ch in enumerate(chars) }  # 编码: 'a' → 1, 'b' → 2, ...
itos = { i:ch for i, ch in enumerate(chars) }  # 解码: 1 → 'a', 2 → 'b', ...
BOS = stoi['<BOS>']  # = 0
💡 为什么需要 <BOS> 特殊标记？
BOS (Beginning Of Sequence) 用于标识序列的开始和结束。训练时，每个名字两端都加上 BOS：
"emma" → [BOS, e, m, m, a, BOS]
• 开头的 BOS 告诉模型"从这里开始生成"
• 结尾的 BOS 让模型学会"在这里停止"
推理时，模型从 BOS 开始，生成到 BOS 就停止。
📊 对比：真实 LLM 的分词器
MicroGPT 用字符级分词（vocab=27），而 GPT-2 使用 BPE 分词（vocab=50,257），ChatGPT 使用更大的词表。
字符级分词简单但序列更长；BPE 分词更高效但实现更复杂。

3 自动微分引擎 / Autograd Engine

这是整个程序最精妙的部分。Value 类让每个标量值都能自动追踪梯度，是 PyTorch 等深度学习框架的核心原理。

3.1 Value 类的核心结构

每个 Value 对象存储 4 个关键信息：

class Value:
    def __init__(self, data, _children=(), _op=''):
        self.data = data       # 实际数值（前向传播的结果）
        self.grad = 0          # 梯度（反向传播时计算）
        self._backward = lambda: None  # 闭包：如何把梯度传给父节点
        self._prev = set(_children)    # 产生自己的那些节点（计算图的边）
        self._op = _op         # 运算符，用于调试

3.2 运算符重载 —— 构建计算图

每次数学运算都会自动构建计算图。以乘法为例：

def __mul__(self, other):
    other = other if isinstance(other, Value) else Value(other)
    out = Value(self.data * other.data, (self, other), '*')  # ① 前向计算
    def _backward():
        self.grad += other.data * out.grad   # ② 定义反向传播规则
        other.grad += self.data * out.grad   #    d(a*b)/da = b, d(a*b)/db = a
    out._backward = _backward
    return out
🔑 关键洞察 / Key Insight
每次运算做三件事：
1. 前向计算：算出结果值
2. 记录来源：记住"谁产生了我"（构建计算图）
3. 定义反向规则：用闭包保存"如何把梯度传回去"
这就是自动微分的全部秘密！

3.3 支持的运算及其梯度

运算 Op前向 Forward反向 Backward (∂L/∂x)数学直觉
x + yx + ygrad 直接传递加法不改变斜率
x * yx × yy × grad / x × grad乘法的导数是对方
x ** nx^nn·x^(n-1) × grad幂函数求导法则
log(x)ln(x)(1/x) × grad对数的导数是倒数
exp(x)e^xe^x × grad指数函数导数是自身
relu(x)max(0, x)(x>0) × grad正数通过，负数截断

3.4 反向传播 —— 链式法则的自动化

def backward(self):
    # Step 1: 拓扑排序 —— 找到所有节点的正确计算顺序
    topo = []
    visited = set()
    def build_topo(v):
        if v not in visited:
            visited.add(v)
            for child in v._prev:
                build_topo(child)
            topo.append(v)
    build_topo(self)

    # Step 2: 从 loss 开始，逆序传播梯度
    self.grad = 1  # dL/dL = 1
    for v in reversed(topo):
        v._backward()  # 每个节点把梯度传给自己的父节点
🧮 链式法则直觉 / Chain Rule Intuition
如果 L = f(g(x))，那么 dL/dx = dL/df · df/dg · dg/dx

就像问"最终结果对 x 有多敏感？"—— 把每一步的敏感度乘起来。

例子：L = log(a*b + 4)，其中 a=2, b=3
• dL/d(a*b+4) = 1/10 = 0.1 （log 的导数）
• d(a*b+4)/d(a*b) = 1 （加法的导数）
• d(a*b)/da = b = 3 （乘法对 a 的导数）
• 所以 dL/da = 0.1 × 1 × 3 = 0.3 ✓

4.2 参数矩阵 / Parameter Matrices

# 创建随机初始化的参数矩阵
matrix = lambda nout, nin, std=0.02: [
    [Value(random.gauss(0, std)) for _ in range(nin)]
    for _ in range(nout)
]

state_dict = {
    'wte': matrix(vocab_size, n_embd),   # Token 嵌入表: 27×16
    'wpe': matrix(block_size, n_embd),   # 位置嵌入表: 8×16
    'lm_head': matrix(vocab_size, n_embd), # 输出投影: 27×16
}

# 每层 Transformer 的参数
for i in range(n_layer):
    state_dict[f'layer{i}.attn_wq'] = matrix(n_embd, n_embd)  # Q 投影: 16×16
    state_dict[f'layer{i}.attn_wk'] = matrix(n_embd, n_embd)  # K 投影: 16×16
    state_dict[f'layer{i}.attn_wv'] = matrix(n_embd, n_embd)  # V 投影: 16×16
    state_dict[f'layer{i}.attn_wo'] = matrix(n_embd, n_embd, std=0)  # 输出投影 (初始化为0附近)
    state_dict[f'layer{i}.mlp_fc1'] = matrix(4*n_embd, n_embd)  # MLP 扩展: 64×16
    state_dict[f'layer{i}.mlp_fc2'] = matrix(n_embd, 4*n_embd, std=0)  # MLP 压缩: 16×64
💡 为什么 attn_wo 和 mlp_fc2 初始化为 ~0？
这两个矩阵的 std=0 意味着初始值接近零。因为它们的输出会通过残差连接加回输入：
x = x_residual + output
初始时 output ≈ 0，所以 x ≈ x_residual，相当于"什么都不做"。
这让训练初期更稳定，模型可以逐渐学会有用的变换。

5 GPT 模型架构 / GPT Model Architecture

GPT 的前向传播是一个无状态函数：输入 token 序列和参数，输出下一个 token 的概率分布。

5.1 辅助函数 / Helper Functions

Linear（线性变换）

def linear(x, w):
    # 矩阵乘法: y = W · x
    # x: 输入向量 [n_in], w: 权重矩阵 [n_out × n_in]
    return [sum(wi * xi for wi, xi in zip(wo, x)) for wo in w]

# 例: linear([1,2,3], [[1,0,0],[0,1,0]]) = [1, 2]  (3维→2维)

Softmax（概率归一化）

def softmax(logits):
    max_val = max(val.data for val in logits)  # 数值稳定性技巧
    exps = [(val - max_val).exp() for val in logits]
    total = sum(exps)
    return [e / total for e in exps]

# 把任意实数变成概率分布（所有值 ∈ [0,1]，总和 = 1）

🎮 交互演示：Softmax

RMSNorm（归一化）

def rmsnorm(x):
    ms = sum(xi * xi for xi in x) / len(x)  # 均方值 (Mean Square)
    scale = (ms + 1e-5) ** -0.5              # 1 / √(ms + ε)
    return [xi * scale for xi in x]          # 缩放每个元素
🔑 RMSNorm 的作用
让向量的"能量"（RMS 值）归一化到约 1.0，防止数值在深层网络中爆炸或消失。
相比 LayerNorm，RMSNorm 省去了均值计算和偏移参数，更简单高效。
参考：Root Mean Square Layer Normalization (论文)

5.2 嵌入层 / Embedding Layer

def gpt(token_id, pos_id, keys, values):
    tok_emb = state_dict['wte'][token_id]  # 查表: token → 16维向量
    pos_emb = state_dict['wpe'][pos_id]    # 查表: 位置 → 16维向量
    x = [t + p for t, p in zip(tok_emb, pos_emb)]  # 逐元素相加
    x = rmsnorm(x)  # 归一化
💡 嵌入的直觉 / Embedding Intuition
想象一个 16 维空间：
• 每个字符被映射到这个空间中的一个点（Token Embedding）
• 每个位置也被映射到一个点（Position Embedding）
• 两者相加 = 模型同时知道"这是什么字符"和"它在第几个位置"

训练过程中，相似的字符（如元音 a, e, i）会在嵌入空间中靠近彼此。

5.3 多头注意力 / Multi-Head Attention

这是 Transformer 的核心，让模型能"关注"序列中不同位置的信息。

# Step 1: 计算 Q, K, V
q = linear(x, state_dict[f'layer{li}.attn_wq'])  # Query: "我在找什么？"
k = linear(x, state_dict[f'layer{li}.attn_wk'])  # Key:   "我能提供什么？"
v = linear(x, state_dict[f'layer{li}.attn_wv'])  # Value: "我的实际内容"

# 缓存 K, V 用于后续 token（KV Cache）
keys[li].append(k)
values[li].append(v)

# Step 2: 对每个注意力头
for h in range(n_head):  # 4 个头
    hs = h * head_dim    # 每个头处理 4 维
    q_h = q[hs:hs+head_dim]
    k_h = [ki[hs:hs+head_dim] for ki in keys[li]]   # 所有历史 K
    v_h = [vi[hs:hs+head_dim] for vi in values[li]]  # 所有历史 V

    # Step 3: 计算注意力分数 = Q·K^T / √d
    attn_logits = [sum(q_h[j] * k_h[t][j] for j in range(head_dim))
                   / head_dim**0.5
                   for t in range(len(k_h))]

    # Step 4: Softmax → 概率分布
    attn_weights = softmax(attn_logits)

    # Step 5: 加权求和 V
    head_out = [sum(attn_weights[t] * v_h[t][j]
                    for t in range(len(v_h)))
                for j in range(head_dim)]

# Step 6: 拼接所有头 + 输出投影 + 残差连接
x = linear(x_attn, state_dict[f'layer{li}.attn_wo'])
x = [a + b for a, b in zip(x, x_residual)]  # 残差连接
🧩 多头注意力的直觉 / Multi-Head Intuition
4 个头就像 4 个"专家"，各自关注不同的特征：
• 头 1 可能关注：前一个字符是什么（局部模式）
• 头 2 可能关注：名字的开头字母（全局模式）
• 头 3 可能关注：元音/辅音的交替模式
• 头 4 可能关注：名字的长度模式

最后把 4 个头的输出拼接起来，得到更丰富的表示。
参考：Attention Is All You Need (原始论文)

5.4 MLP 前馈网络 / Feed-Forward Network

# 注意力之后，每个位置独立地通过 MLP
x_residual = x
x = rmsnorm(x)
x = linear(x, state_dict[f'layer{li}.mlp_fc1'])  # 16 → 64 (扩展 4 倍)
x = [xi.relu() ** 2 for xi in x]                  # ReLU² 激活函数
x = linear(x, state_dict[f'layer{li}.mlp_fc2'])  # 64 → 16 (压缩回来)
x = [a + b for a, b in zip(x, x_residual)]        # 残差连接
🔑 MLP 的作用
注意力机制负责"信息交换"（不同位置之间），MLP 负责"信息处理"（每个位置独立）。

ReLU² vs ReLU：relu(x)² = max(0, x)²
• 普通 ReLU 在 x=0 处不可导（有尖角）
• ReLU² 在 x=0 处平滑过渡，梯度更好
• 这是 Primer 论文 中发现的改进

    # 最后一步：把 16 维向量映射回词表大小（27）
    logits = linear(x, state_dict['lm_head'])

6 训练循环 / Training Loop

6.1 数据准备 / Data Preparation

doc = docs[step % len(docs)]                       # 循环取一个名字
tokens = [BOS] + [stoi[ch] for ch in doc] + [BOS]  # 分词 + 加 BOS
n = min(block_size, len(tokens) - 1)                # 截断到最大长度 8

# 例: "emma" → tokens = [0, 5, 13, 13, 1, 0], n = 5

6.2 前向传播与损失计算 / Forward Pass & Loss

keys, values = [[] for _ in range(n_layer)], [[] for _ in range(n_layer)]
losses = []
for pos_id in range(n):
    token_id = tokens[pos_id]      # 当前 token
    target_id = tokens[pos_id + 1] # 下一个 token（标签）

    logits = gpt(token_id, pos_id, keys, values)  # 模型预测
    probs = softmax(logits)                         # 转为概率
    loss_t = -probs[target_id].log()                # 交叉熵损失
    losses.append(loss_t)

loss = (1 / n) * sum(losses)  # 平均损失
💡 交叉熵损失的直觉 / Cross-Entropy Intuition
loss = -log(P(正确答案))

• 模型给正确答案 90% 概率 → loss = -log(0.9) = 0.105 ✅ 低损失
• 模型给正确答案 10% 概率 → loss = -log(0.1) = 2.302 ❌ 高损失
• 模型给正确答案 1% 概率 → loss = -log(0.01) = 4.605 ❌❌ 很高

损失越低 = 模型预测越准确。训练的目标就是让损失尽可能低。

6.3 反向传播 / Backward Pass

loss.backward()  # 一行代码，自动计算所有 ~5000 个参数的梯度！

这就是第 3 节 Autograd 引擎的威力。backward() 会沿着计算图逆序遍历，用链式法则自动计算每个参数的梯度。

6.4 Adam 优化器 / Adam Optimizer

# 学习率线性衰减: 从 0.01 逐渐降到 0
lr_t = learning_rate * (1 - step / num_steps)

for i, p in enumerate(params):
    # 一阶动量 (方向): 梯度的指数移动平均
    m[i] = beta1 * m[i] + (1 - beta1) * p.grad          # β1 = 0.9

    # 二阶动量 (步长): 梯度平方的指数移动平均
    v[i] = beta2 * v[i] + (1 - beta2) * p.grad ** 2     # β2 = 0.95

    # 偏差修正 (训练初期 m,v 偏向 0)
    m_hat = m[i] / (1 - beta1 ** (step + 1))
    v_hat = v[i] / (1 - beta2 ** (step + 1))

    # 更新参数
    p.data -= lr_t * m_hat / (v_hat ** 0.5 + eps_adam)

    # 梯度清零，为下一步做准备
    p.grad = 0
🧮 Adam vs SGD 的直觉
SGD（随机梯度下降）：每步沿梯度方向走固定步长
→ 问题：在平坦区域走太慢，在陡峭区域走太快

Adam：自适应调整每个参数的步长
• m (一阶动量) = 梯度的"惯性"，让方向更稳定（不会来回震荡）
• v (二阶动量) = 自适应步长，梯度大的参数走小步，梯度小的走大步
• 偏差修正 = 训练初期 m,v 都是 0，会低估真实值，需要修正

参考：Adam: A Method for Stochastic Optimization (原始论文)

temperature = 0.6  # 控制"创意"程度

for sample_idx in range(20):
    keys, values = [[] for _ in range(n_layer)], [[] for _ in range(n_layer)]
    token_id = BOS  # 从 BOS 开始

7.1 Temperature 温度参数

7.2 KV Cache

keys[li].append(k)    # 缓存当前 token 的 Key
values[li].append(v)  # 缓存当前 token 的 Value
⚡ KV Cache 的作用
注意力机制需要看到所有之前的 token。如果每次都重新计算所有 K 和 V，计算量是 O(n²)。

KV Cache 的思路：之前 token 的 K 和 V 不会变，缓存起来复用即可。
• 第 1 个 token：计算 K₁, V₁，缓存
• 第 2 个 token：计算 K₂, V₂，缓存。注意力用 [K₁,K₂] 和 [V₁,V₂]
• 第 n 个 token：只需计算 Kₙ, Vₙ，注意力用 [K₁,...,Kₙ] 和 [V₁,...,Vₙ]

这是 LLM 推理加速的核心技巧，也是为什么 GPU 显存是推理的瓶颈。
