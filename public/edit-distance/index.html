1 什么是编辑距离

编辑距离（Levenshtein Distance）衡量将字符串 A 变成字符串 B 所需的最少操作次数。

三种允许的操作

插入
添加一个字符
删除
移除一个字符
替换
换成另一个字符
🔤

直觉理解

想象你在文本编辑器里，只能用「退格删除」「打字插入」「选中替换」三个键，把一个单词改成另一个单词——最少按几次键？这就是编辑距离。

举个例子

AB操作编辑距离
abcadc替换 b→d1
kittensitting替换 k→s, 替换 e→i, 插入 g3
abcabc无需操作0
abc删除 3 次3

2 动态规划求解

状态定义

📐 dp[i][j] 的含义

构建 (m+1) × (n+1) 的矩阵，dp[i][j] = A 的前 i 个字符变成 B 的前 j 个字符的最小操作数。

初始化

边界含义值
dp[i][0] = iA 的前 i 个字符 → 空串删除 i 次
dp[0][j] = j空串 → B 的前 j 个字符插入 j 次

转移方程

🧮 核心公式

如果 A[i] == B[j]（字符相同）：

dp[i][j] = dp[i-1][j-1]    // 无需操作，直接继承

如果 A[i] != B[j]（字符不同）：

dp[i][j] = 1 + min(
    dp[i-1][j],      // ← 删除 A[i]
    dp[i][j-1],      // ← 插入 B[j]
    dp[i-1][j-1]     // ← 替换 A[i] 为 B[j]
)

三个方向的含义

⬆️ 上方 dp[i-1][j]
删除 A[i]
⬅️ 左方 dp[i][j-1]
插入 B[j]
↖️ 左上 dp[i-1][j-1]
替换 或 匹配
💡 一句话总结

每个格子在问：「到这一步为止，最少改几次？」——从上面删除(+1)、从左边插入(+1)、从左上角替换(不同+1，相同+0)，三选最小。

3 手推示例：abc → adc

第一步：初始化边界

第 0 行和第 0 列表示「从空串变过去需要多少步」：

📖 读法

dp[0][2]=2：空串变成 "ad" 需要插入 2 次。dp[3][0]=3："abc" 变成空串需要删除 3 次。

第二步：逐格填写

dp[1][1]：A[1]=a vs B[1]=a → 相同，取左上角 dp[0][0]=0

dp[1][2]：A[1]=a vs B[2]=d → 不同，min(左0, 上2, 左上1)+1 = 1

dp[1][3]：A[1]=a vs B[3]=c → 不同，min(左1, 上3, 左上2)+1 = 2

dp[2][1]：A[2]=b vs B[1]=a → 不同，min(左2, 上0, 左上1)+1 = 1

dp[2][2]：A[2]=b vs B[2]=d → 不同，min(左1, 上1, 左上0)+1 = 1

dp[2][3]：A[2]=b vs B[3]=c → 不同，min(左1, 上2, 左上1)+1 = 2

dp[3][1]：A[3]=c vs B[1]=a → 不同，min(左3, 上1, 左上1)+1 = 2

dp[3][2]：A[3]=c vs B[2]=d → 不同，min(左2, 上1, 左上1)+1 = 2

dp[3][3]：A[3]=c vs B[3]=c → 相同，取左上角 dp[2][2]=1

最终矩阵

""0123

b2112
c3221
✅ 结果

右下角 dp[3][3] = 1，即 abc → adc 只需 1 步（把 b 替换成 d）。

5 编辑距离转相似度

📐 公式

similarity = 1 - edit_distance / max(len(A), len(B))

文件名 A文件名 B编辑距离相似度
report_v1.txtreport_v2.txt1≈ 0.93 ✅
readme.mdreadme.txt30.70
readme.mdlicense.txt9≈ 0.18 ❌
💡 阈值建议

通常 similarity > 0.7 可认为「相似文件名」，具体按场景调整。

6 Python 实现

空间优化版（滚动数组）

def edit_distance(a: str, b: str) -> int:
    m, n = len(a), len(b)
    prev = list(range(n + 1))
    curr = [0] * (n + 1)
    for i in range(1, m + 1):
        curr[0] = i
        for j in range(1, n + 1):
            if a[i-1] == b[j-1]:
                curr[j] = prev[j-1]
            else:
                curr[j] = 1 + min(prev[j], curr[j-1], prev[j-1])
        prev, curr = curr, prev
    return prev[n]

相似度计算

def filename_similarity(a: str, b: str) -> float:
    dist = edit_distance(a, b)
    max_len = max(len(a), len(b))
    return 1 - dist / max_len if max_len > 0 else 1.0

复杂度

维度复杂度说明
时间O(m × n)两层循环
空间O(min(m, n))滚动数组优化
