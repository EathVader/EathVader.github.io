1为什么需要 B+ 树？

数据库和文件系统需要高效地存储和检索大量数据。二叉搜索树（BST）在数据量大时树高过大，导致大量磁盘 I/O。

数据结构树高 (100万数据)磁盘 I/O 次数适用场景
二叉搜索树~20 层~20 次内存数据
AVL / 红黑树~20 层~20 次内存数据
B+ 树 (阶数=100)~3 层~3 次磁盘存储
💡 核心优势

B+ 树是多路平衡搜索树，每个节点可以有多个子节点（通常 100+），树高很低。对于磁盘存储，每次 I/O 读取一个节点（通常 4KB），树高越低，I/O 越少。

2B+ 树的结构

B+ 树的特点：

  • 所有数据都存储在叶子节点
  • 内部节点只存储键值和子节点指针（索引）
  • 叶子节点通过链表连接，支持范围查询
  • 所有叶子节点在同一层
  • 每个节点最多有 m 个子节点（m 为阶数）
B+ 树：内部节点只存索引，数据全在叶子节点，叶子节点用链表连接

3磁盘存储 vs 内存存储

磁盘 B+ 树与内存 B+ 树的关键区别：

特性内存 B+ 树磁盘 B+ 树
节点大小动态分配固定大小（通常 4KB 页）
节点标识内存指针磁盘偏移量（offset）
缓存无需缓存需要 Buffer Pool 缓存热点页
序列化无需节点需要序列化/反序列化
持久化重启丢失写入磁盘，重启可恢复

# 磁盘 B+ 树的节点表示
PAGE_SIZE = 4096  # 4KB 页大小

class DiskNode:
    def __init__(self, is_leaf=False, order=32):
        self.is_leaf = is_leaf
        self.order = order
        self.keys = []          # 键列表
        self.values = []        # 值列表（仅叶子节点）
        self.children = []      # 子节点偏移量（仅内部节点）
        self.next_leaf = None   # 下一个叶子节点偏移量

    def to_bytes(self):
        """序列化节点到字节（写入磁盘）"""
        data = struct.pack('?I', self.is_leaf, len(self.keys))
        for key in self.keys:
            data += struct.pack('q', key)  # 8 字节整数
        if self.is_leaf:
            for val in self.values:
                data += struct.pack('q', val)
        else:
            for child in self.children:
                data += struct.pack('q', child)  # 子节点偏移量
        # 填充到 PAGE_SIZE
        data += b'\x00' * (PAGE_SIZE - len(data))
        return data

    @classmethod
    def from_bytes(cls, data):
        """从字节反序列化节点（从磁盘读取）"""
        is_leaf, key_count = struct.unpack('?I', data[:5])
        node = cls(is_leaf=is_leaf)
        offset = 5
        for _ in range(key_count):
            key = struct.unpack('q', data[offset:offset+8])[0]
            node.keys.append(key)
            offset += 8
        # ... 反序列化 values 和 children
        return node

4插入操作与节点分裂

插入操作的流程：

  1. 从根节点开始，找到合适的叶子节点
  2. 将键值插入叶子节点（保持有序）
  3. 如果叶子节点溢出（键数 > order-1），进行节点分裂
  4. 分裂可能导致父节点溢出，递归向上分裂
  5. 如果根节点分裂，创建新的根节点，树高 +1
节点分裂：溢出节点分裂为两个，中间键上移到父节点

5查找操作

B+ 树的查找非常高效：

Python
class BPlusTree:
    def search(self, key):
        """查找键对应的值"""
        node = self.root
        while not node.is_leaf:
            # 在内部节点中找到合适的子节点
            i = 0
            while i < len(node.keys) and key >= node.keys[i]:
                i += 1
            node = self.load_node(node.children[i])  # 从磁盘加载
        # 在叶子节点中查找
        for i, k in enumerate(node.keys):
            if k == key:
                return node.values[i]  # 找到！
        return None  # 未找到

    def range_query(self, start, end):
        """范围查询：利用叶子节点的链表"""
        results = []
        node = self._find_leaf(start)
        while node:
            for i, k in enumerate(node.keys):
                if start <= k <= end:
                    results.append((k, node.values[i]))
                elif k > end:
                    return results
            node = self.load_node(node.next_leaf) if node.next_leaf else None
        return results
✅ 范围查询优势

B+ 树的叶子节点通过链表连接，范围查询只需找到起始位置，然后沿链表遍历即可。这是 B+ 树相比 B 树的主要优势。

6Python 实现要点

Python
import struct
import os

class DiskBPlusTree:
    def __init__(self, db_path, order=32):
        self.db_path = db_path
        self.order = order
        self.page_size = 4096
        self.root_offset = 0
        self.free_list = []  # 空闲页列表
        if not os.path.exists(db_path):
            self._create_new_tree()
        else:
            self._load_metadata()

    def _create_new_tree(self):
        """创建新的 B+ 树文件"""
        with open(self.db_path, 'wb') as f:
            # 页 0: 元数据
            meta = struct.pack('qqq', self.order, self.root_offset, len(self.free_list))
            f.write(meta.ljust(self.page_size, b'\x00'))
            # 页 1: 根节点（初始为空叶子）
            root = self._serialize_node(is_leaf=True, keys=[], values=[], next_leaf=0)
            f.write(root)
        self.root_offset = self.page_size

    def _serialize_node(self, is_leaf, keys, values=None, children=None, next_leaf=0):
        """序列化节点"""
        data = struct.pack('?I', is_leaf, len(keys))
        for k in keys:
            data += struct.pack('q', k)
        if is_leaf:
            for v in (values or []):
                data += struct.pack('q', v)
            data += struct.pack('q', next_leaf)
        else:
            for c in (children or []):
                data += struct.pack('q', c)
        return data.ljust(self.page_size, b'\x00')

    def insert(self, key, value):
        """插入键值对"""
        # 1. 找到叶子节点
        path = self._find_path(key)
        leaf = self._load_node(path[-1])
        # 2. 插入键值
        idx = self._bisect_left(leaf.keys, key)
        leaf.keys.insert(idx, key)
        leaf.values.insert(idx, value)
        # 3. 检查是否需要分裂
        if len(leaf.keys) > self.order - 1:
            self._split_leaf(path, leaf)

    def _split_leaf(self, path, leaf):
        """分裂叶子节点"""
        mid = len(leaf.keys) // 2
        # 创建新右节点
        right_keys = leaf.keys[mid:]
        right_values = leaf.values[mid:]
        right_next = leaf.next_leaf
        # 更新左节点
        leaf.keys = leaf.keys[:mid]
        leaf.values = leaf.values[:mid]
        leaf.next_leaf = self._allocate_page()
        # 写入右节点
        right_data = self._serialize_node(True, right_keys, right_values, next_leaf=right_next)
        self._write_page(leaf.next_leaf, right_data)
        # 上移中间键
        self._insert_into_parent(path, leaf, right_keys[0], leaf.next_leaf)

    def _allocate_page(self):
        """分配新页"""
        if self.free_list:
            return self.free_list.pop()
        # 追加到文件末尾
        return os.path.getsize(self.db_path)

    def _write_page(self, offset, data):
        """写入页到磁盘"""
        with open(self.db_path, 'r+b') as f:
            f.seek(offset)
            f.write(data)
💡 实现要点
  • 固定页大小：每个节点占用固定大小的页（4KB），方便磁盘寻址
  • 偏移量代替指针：用文件偏移量（offset）代替内存指针
  • 序列化/反序列化：节点需要序列化为字节写入磁盘
  • 空闲页管理：删除节点后的页加入空闲列表，复用空间
  • Buffer Pool：生产环境需要缓存热点页，减少磁盘 I/O

8总结

特性说明
多路平衡每个节点有多个子节点，树高很低
磁盘友好节点大小 = 磁盘页大小，一次 I/O 读一个节点
范围查询叶子节点链表连接，范围查询高效
节点分裂插入溢出时分裂节点，中间键上移
磁盘实现用偏移量代替指针，序列化节点到固定页
✅ 核心要点
  • B+ 树是数据库索引的核心数据结构
  • 所有数据在叶子节点，内部节点只做索引
  • 磁盘实现需要固定页大小、偏移量寻址、序列化
  • 叶子节点链表支持高效的范围查询
  • Python 实现可以用 struct 模块序列化节点
