1 URL 解析

📝

类比：写信封

输入 URL 后，浏览器首先解析 URL，就像写信封一样：

  • 协议（scheme）：用什么方式寄信（HTTP/HTTPS）
  • 域名（hostname）：收件人地址（google.com）
  • 端口（port）：找哪个门卫（默认 80/443）
  • 路径（path）：具体楼层（/index.html）
  • 参数（query）：信的内容（?id=123）
📝 URL 格式
protocol://hostname:port/path?query#fragment

例如：https://www.google.com:443/search?q=hello#results

解析过程：

  1. 检查协议，如果是 http 补全为 http://，https 补全为 https://
  2. 提取域名、端口、路径、参数
  3. 检查缓存（浏览器缓存、操作系统缓存）
  4. 如果缓存未命中，发起网络请求

2 DNS 解析

浏览器需要将域名转换为 IP 地址：

💡 DNS 缓存层级
  1. 浏览器 DNS 缓存（TTL 约 1 分钟）
  2. 操作系统 DNS 缓存（TTL 约 1 小时）
  3. 本地 DNS 服务器（ISP DNS，TTL 几小时）
  4. 根服务器 → TLD 服务器 → 权威 DNS

3 TCP 连接

获取 IP 后，浏览器与服务器建立 TCP 连接（如果是 HTTPS 则还有 TLS）：

📞

类比：打电话

  • 你拨通号码（发送 SYN）
  • 对方接听并确认（返回 SYN+ACK）
  • 你说"好的"（发送 ACK）
  • 现在开始通话（TCP 连接建立）

4 TLS 握手 (HTTPS)

如果使用 HTTPS，还需要进行 TLS 握手来建立加密通道：

📝 TLS 握手关键步骤
  1. ClientHello：客户端发送支持的密码套件列表、随机数
  2. ServerHello：服务器选择密码套件、发送证书、服务器随机数
  3. 证书验证：客户端验证服务器证书（是否过期、是否可信）
  4. 密钥交换：客户端生成预主密钥，用服务器公钥加密发送
  5. 会话密钥生成：双方用预主密钥 + 随机数生成会话密钥
⚠️ HTTPS 跳转

如果输入的是 http://google.com，服务器会返回 301/302 重定向，让浏览器跳转到 https:// 版本。

5 HTTP 请求/响应

TCP/TLS 连接建立后，浏览器发送 HTTP 请求，服务器返回响应：

HTTP/2 和 HTTP/3

🚀 性能优化
  • HTTP/1.1：持久连接、管道化（但串行处理）
  • HTTP/2：多路复用、头部压缩、服务器推送
  • HTTP/3：QUIC 协议（UDP）、无队头阻塞、0-RTT

8 深入分析：真实世界的细节

HTTP → HTTPS 重定向

如果你输入的是 http://google.com，实际会发生两次重定向：

🔄 重定向链
  1. 301 永久重定向：主机名规范化，google.com → www.google.com
  2. 307 内部重定向：HTTP → HTTPS，浏览器内部将协议改为 https://，端口改为 443

两次重定向后，浏览器才真正发起 HTTPS 请求。

负载均衡与 CDN

Google.com 不只对应一个 IP 地址。DNS 可能返回多个 IP，浏览器默认选择第一个。请求到达 Google 的负载均衡器后：

  1. WAF 检查：Web 应用程序防火墙检查是否为恶意请求
  2. 安全连接终止：TLS 在负载均衡器处解密（内部流量不需要加密）
  3. CDN 分发：请求被分配到最近的 CDN 节点，返回预压缩的页面

HTTP/2 并发请求

Google 主页优化后包含预渲染内容、内联 CSS/JS 和图片。首次加载后会触发一系列并发请求（HTTP/2 多路复用）：

  • 下载字体、Logo 图像、用户头像
  • JavaScript 解析（可能使用 defer 或 async 非阻塞模式）
  • Cookie/OAuth 令牌认证，告诉服务器"我是谁"

User-Agent 嗅探

服务器端（而非客户端）会检测你的浏览器类型。如果你不是用 Chrome 访问 Google，他们可能会弹出提示推荐使用 Chrome。这说明 User-Agent 嗅探是在服务器端进行的。

缓存策略

浏览器根据响应头的缓存策略对内容进行缓存。Google 可能对静态资源设置了较长的缓存时间，对动态内容设置了较短的缓存时间或 no-cache。

📊 实际抓包观察

在浏览器中打开 google.com 并禁用缓存，你会看到：

  • 301 状态码：主机名规范化
  • 307 内部重定向：HTTP → HTTPS
  • 字体、Logo、头像等资源的下载
  • 没有 API 调用，说明个人信息是服务器端推送的
