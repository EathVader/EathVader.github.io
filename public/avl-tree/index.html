1 什么是 AVL 树 / What is AVL Tree

⚖️

类比 / Analogy

AVL 树就像一个强迫症整理师：每次放入或取出一本书后，都要确保书架左右两边的高度差不超过 1。虽然整理花时间，但找书永远很快。

AVL 树由苏联数学家 Adelson-Velsky 和 Landis 于 1962 年发明，是最早的自平衡二叉搜索树。核心性质：

📐 AVL 树的平衡条件

对于任意节点，其左子树高度与右子树高度之差的绝对值 ≤ 1。

这比红黑树更严格（红黑树允许最长路径是最短的 2 倍），所以 AVL 树更矮、查找更快。

左：所有节点 |bf| ≤ 1 ✅　右：根节点 bf=2，失衡 ❌

2 高度与平衡因子 / Height & Balance Factor

📏 定义
  • 节点高度 = 到最远叶节点的边数（叶节点高度=0，空节点高度=-1）
  • 平衡因子 (bf) = 左子树高度 − 右子树高度
  • AVL 树要求：所有节点 |bf| ≤ 1
// 获取高度
int height(Node node) {
    return node == null ? -1 : node.height;
}

// 平衡因子
int balanceFactor(Node node) {
    return height(node.left) - height(node.right);
}

// 更新高度
void updateHeight(Node node) {
    node.height = max(height(node.left), height(node.right)) + 1;
}

AVL 树的高度严格保证在 log₂(n+1) 到 1.44·log₂(n+2) 之间，比红黑树（最高 2·log₂(n+1)）更矮。

3 四种旋转 / Four Rotations

当插入/删除导致某节点 |bf| = 2 时，需要旋转恢复平衡。根据失衡方向，分为 4 种情况：

失衡类型条件旋转方式记忆口诀
LLbf=2，左孩子 bf≥0右旋左左 → 右旋
LRbf=2，左孩子 bf<0先左旋左孩子，再右旋左右 → 先左后右
RRbf=-2，右孩子 bf≤0左旋右右 → 左旋
RLbf=-2，右孩子 bf>0先右旋右孩子，再左旋右左 → 先右后左

LL 型：右旋

LR 型：先左旋再右旋

LR 型：C 的左孩子 A 的右孩子 B 导致失衡 → 先左旋 A → 变成 LL 型 → 再右旋 C

RL 型：先右旋再左旋

RL 型：A 的右孩子 C 的左孩子 B 导致失衡 → 先右旋 C → 变成 RR 型 → 再左旋 A
// 平衡操作
Node balance(Node node) {
    updateHeight(node);
    int bf = balanceFactor(node);
    if (bf == 2) {                          // 左偏
        if (balanceFactor(node.left) < 0)   // LR 型
            node.left = leftRotate(node.left);
        return rightRotate(node);           // LL 型
    }
    if (bf == -2) {                         // 右偏
        if (balanceFactor(node.right) > 0)  // RL 型
            node.right = rightRotate(node.right);
        return leftRotate(node);            // RR 型
    }
    return node;
}

4 插入与平衡 / Insertion & Rebalancing

插入 = BST 标准插入 + 回溯时逐层平衡。

Node insert(Node node, int val) {
    if (node == null) return new Node(val);
    if (val < node.val)
        node.left = insert(node.left, val);
    else
        node.right = insert(node.right, val);
    return balance(node);  // 回溯时平衡！
}
💡 关键点

插入最多只需要一次旋转（单旋或双旋）就能恢复平衡。因为一次旋转后子树高度恢复原值，不会继续向上传播失衡。

5 删除与平衡 / Deletion & Rebalancing

删除 = BST 标准删除 + 回溯时逐层平衡。与插入不同，删除可能需要多次旋转（最多 O(log n) 次）。

Node delete(Node node, int val) {
    if (node == null) return null;
    if (val < node.val)
        node.left = delete(node.left, val);
    else if (val > node.val)
        node.right = delete(node.right, val);
    else {
        // 找到目标节点
        if (node.left == null) return node.right;
        if (node.right == null) return node.left;
        // 两个孩子：用右子树最小值替换
        Node min = findMin(node.right);
        node.val = min.val;
        node.right = delete(node.right, min.val);
    }
    return balance(node);  // 回溯时平衡！
}
⚠️ 插入 vs 删除的旋转次数

插入：最多 1 次旋转（旋转后高度恢复，不再传播）
删除：最多 O(log n) 次旋转（旋转后高度可能降低，继续向上传播）

6 AVL vs 红黑树 / AVL vs Red-Black Tree

AVL 树红黑树
平衡条件|bf| ≤ 1（严格）黑色平衡（宽松）
最大高度1.44·log n2·log n
查找⚡ 更快（树更矮）稍慢
插入旋转≤ 1 次≤ 2 次
删除旋转≤ O(log n) 次≤ 3 次
适用场景读多写少（数据库索引）写多读少（内核、容器库）
🏃‍♂️

一句话总结

AVL 树：查找快，但插入删除时"整理"更频繁 → 适合读密集场景
红黑树：查找稍慢，但"整理"更轻量 → 适合写密集场景
