2 推模式（写扩散）

原理

用户发布消息时，主动推送到所有粉丝的收件箱。

A 发布消息
→
找到所有粉丝
→
推送到每个粉丝 Feed
优点缺点
✅ 读取快（直接读收件箱）❌ 写压力大（大V推送量巨大）
✅ 实时性好❌ 存储冗余
📌 适用场景

粉丝数均衡、读多写少、实时性要求高。

3 拉模式（读扩散）

原理

用户读取 Feed 时，主动拉取所有关注者的消息并聚合。

用户请求 Feed
→
获取关注列表
→
拉取+聚合+排序
优点缺点
✅ 写压力小❌ 读取慢（需聚合）
✅ 存储简单❌ 缓存困难
📌 适用场景

强关系社交（朋友圈）、关注数少、读延迟容忍度高。
