1 为什么需要布隆过滤器 / Why

📧

类比 / Analogy

你有 10 亿个注册邮箱，新用户注册时要判断邮箱是否已存在。用 HashSet 存？10 亿 × 20 字节 = 20 GB 内存。用布隆过滤器？只需 ~1.2 GB，代价是偶尔误判（说"存在"但实际不存在），但绝不漏判（说"不存在"就一定不存在）。

2 什么是布隆过滤器 / What

📐 核心结构

一个 m 位的位数组（全部初始化为 0）+ k 个独立的哈希函数。每个哈希函数把输入映射到 [0, m-1] 的一个位置。

🔑 两条铁律

说"不存在"→ 一定不存在（任何一位为 0 就确定不在）
说"存在"→ 可能存在（所有位都为 1，但可能是其他元素设置的）

⚠️ 不支持删除

不能把某一位从 1 改回 0，因为这一位可能是其他元素设置的。删除会导致漏判（false negative）。如果需要删除，用 Counting Bloom Filter（每位用计数器代替 0/1）或 Cuckoo Filter。

📐 误判率公式

插入 n 个元素后，误判率约为：
p ≈ (1 - e^(-kn/m))^k
其中 m = 位数组大小，k = 哈希函数个数，n = 已插入元素数。

6 代码实现 / Implementation

class BloomFilter {
    private BitSet bits;
    private int size;
    private int numHash;

    BloomFilter(int expectedElements, double falsePositiveRate) {
        // 最优参数计算
        this.size = (int) (-expectedElements * Math.log(falsePositiveRate)
                          / (Math.log(2) * Math.log(2)));
        this.numHash = (int) (size * Math.log(2) / expectedElements);
        this.bits = new BitSet(size);
    }

    void add(String item) {
        for (int i = 0; i < numHash; i++)
            bits.set(hash(item, i) % size);
    }

    boolean mightContain(String item) {
        for (int i = 0; i < numHash; i++)
            if (!bits.get(hash(item, i) % size)) return false;
        return true; // 可能存在（有误判概率）
    }

    private int hash(String item, int seed) {
        int h = seed;
        for (char c : item.toCharArray())
            h = h * 31 + c;
        return h & 0x7fffffff;
    }
}

7 Cuckoo Filter 对比 / Comparison

🐦

Cuckoo Filter（布谷鸟过滤器）

像布谷鸟抢占其他鸟的巢穴：每个元素存储一个指纹（fingerprint），如果位置被占，就把原住民"踢走"，让它去找另一个位置。支持删除操作，空间效率比 Bloom Filter 更好。

Bloom FilterCuckoo Filter
查询O(k)O(1)~O(2)
插入O(k)O(1) 均摊
删除❌ 不支持✅ 支持
空间效率好更好（<7%误判率时）
实现复杂度简单中等

8 实际应用 / Applications

🌍 工业级应用
  • Redis：内置 Bloom Filter 模块，用于缓存穿透防护
  • Google Chrome：检查 URL 是否在恶意网站黑名单中
  • HBase / Cassandra：判断数据是否在某个 SSTable 中，避免无效磁盘读取
  • Medium：推荐系统中判断用户是否已读过某篇文章
  • 比特币：SPV 节点用 Bloom Filter 过滤交易
