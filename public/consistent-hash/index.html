1 问题：传统哈希的致命缺陷 / The Problem

假设你有 N 台缓存服务器，最简单的负载均衡方式是：

server = hash(key) % N

这在服务器数量固定时工作得很好。但当一台服务器宕机（N → N-1）：

⚠️ 致命问题 / Fatal Problem
(N-1)/N 的缓存数据需要重新映射！
• 3 台服务器挂 1 台 → 66.7% 缓存失效
• 10 台服务器挂 1 台 → 90% 缓存失效
• 100 台服务器挂 1 台 → 99% 缓存失效

这意味着几乎所有请求都会穿透缓存，直接打到数据库 —— 缓存雪崩！

同样，新增一台服务器（N → N+1）也会导致 N/(N+1) 的数据需要迁移。

🎯 我们需要什么？
一种哈希策略，使得增减服务器时，只影响相邻节点的数据，而不是全局重新分配。
这就是一致性哈希（Consistent Hashing）要解决的问题。

2 核心思想：哈希环 / The Hash Ring

🧥 类比：环形衣柜 / Analogy: Circular Closet
一致性哈希就像分配钥匙到环形衣柜。

想象一个圆形的衣柜，上面标着 0 到 360 度的刻度：
• 每个衣柜管理员（服务器）站在环上的某个位置
• 每把钥匙（数据）也被哈希到环上的某个位置
• 钥匙交给顺时针方向遇到的第一个管理员保管

如果一个管理员离开了，他的钥匙只需要交给下一个管理员 —— 其他人完全不受影响！

一致性哈希的核心是将哈希值空间组织成一个环（0 ~ 2³²-1）：

关键步骤：

  1. 将哈希值空间 [0, 2³²-1] 首尾相连，形成一个环
  2. 将服务器节点通过哈希函数映射到环上
  3. 将数据 key 通过同一哈希函数映射到环上
  4. 数据沿顺时针方向找到的第一个服务器节点，就是它的归属

3 节点映射与数据路由 / Node Mapping

# Mapping example
hash("ServerA") → position 100 on ring
hash("ServerB") → position 200 on ring
hash("ServerC") → position 300 on ring

hash("key1") → position 150  → routes to ServerB (next clockwise)
hash("key2") → position 250  → routes to ServerC
hash("key3") → position 350  → routes to ServerA (wraps around)
💡 为什么这样有效？
当 ServerB 宕机时：
• key1（原本在 ServerB）→ 现在顺时针找到 ServerC
• key2（原本在 ServerC）→ 不受影响
• key3（原本在 ServerA）→ 不受影响

只有原本映射到 ServerB 的数据需要迁移，影响范围 ≈ 1/N！

4 虚拟节点：解决倾斜 / Virtual Nodes

如果只有 3 个物理节点，它们在环上的分布可能很不均匀，导致数据倾斜（某台服务器承担大部分请求）。

解决方案：为每个物理节点创建多个虚拟节点（Virtual Nodes），均匀分布在环上：

# Without virtual nodes: 3 points on ring → uneven distribution
# With 150 virtual nodes per server: 450 points → much more even!

hash("ServerA#1") → position 30
hash("ServerA#2") → position 120
hash("ServerA#3") → position 250
...
hash("ServerA#150") → position 340
🔑 虚拟节点的好处
• 负载均衡：虚拟节点越多，数据分布越均匀
• 异构支持：性能强的服务器可以分配更多虚拟节点
• 平滑迁移：增删节点时，数据迁移更均匀

通常每个物理节点分配 100~200 个虚拟节点。

5 节点增删的影响 / Adding & Removing Servers

5.1 删除节点 / Removing a Server

当 ServerB 宕机时，只有 ServerB 负责的数据（ServerA → ServerB 之间的区间）需要迁移到 ServerC。

5.2 添加节点 / Adding a Server

当新增 ServerD 时，只有 ServerD 前一个节点到 ServerD 之间的数据需要从原节点迁移到 ServerD。

💡 影响范围对比 / Impact Comparison
操作传统取模 hash%N一致性哈希
删除 1 个节点 (N=10)~90% 数据迁移~10% 数据迁移
添加 1 个节点 (N=10)~91% 数据迁移~10% 数据迁移
删除 1 个节点 (N=100)~99% 数据迁移~1% 数据迁移

6 Python 实现 / Python Implementation

6.1 从零实现 / From Scratch

import hashlib
import bisect

class ConsistentHash:
    def __init__(self, nodes=None, replicas=150):
        self.replicas = replicas  # virtual nodes per physical node
        self.ring = {}            # hash → node mapping
        self.sorted_keys = []     # sorted hash values for binary search
        for node in (nodes or []):
            self.add_node(node)

    def _hash(self, key):
        return int(hashlib.md5(key.encode()).hexdigest(), 16)

    def add_node(self, node):
        for i in range(self.replicas):
            vnode_key = f"{node}#{i}"
            h = self._hash(vnode_key)
            self.ring[h] = node
            bisect.insort(self.sorted_keys, h)

    def remove_node(self, node):
        for i in range(self.replicas):
            h = self._hash(f"{node}#{i}")
            del self.ring[h]
            self.sorted_keys.remove(h)

    def get_node(self, key):
        if not self.ring:
            return None
        h = self._hash(key)
        idx = bisect.bisect_right(self.sorted_keys, h)
        if idx == len(self.sorted_keys):
            idx = 0  # wrap around
        return self.ring[self.sorted_keys[idx]]

6.2 使用示例 / Usage

ch = ConsistentHash(["ServerA", "ServerB", "ServerC"])

# Route keys
print(ch.get_node("user:1001"))  # → ServerB
print(ch.get_node("user:1002"))  # → ServerA
print(ch.get_node("session:x")) # → ServerC

# Remove a server — only ~1/3 keys remap
ch.remove_node("ServerB")
print(ch.get_node("user:1001"))  # → ServerC (remapped)
print(ch.get_node("user:1002"))  # → ServerA (unchanged!)

# Add a server — only ~1/4 keys remap
ch.add_node("ServerD")
print(ch.get_node("session:x")) # might remap to ServerD

6.3 使用 hash_ring 库 / Using hash_ring Library

from hash_ring import HashRing

servers = ['192.168.0.246:11212',
           '192.168.0.247:11212',
           '192.168.0.249:11212']

ring = HashRing(servers)
print(ring.get_node('my_key'))       # → '192.168.0.247:11212'
print(ring.get_node('another_key'))  # → '192.168.0.249:11212'

8 总结与应用 / Summary & Applications

8.1 核心要点

概念说明
哈希环将哈希空间 [0, 2³²-1] 首尾相连形成环
顺时针路由数据沿顺时针找到第一个服务器节点
虚拟节点每个物理节点映射 100~200 个虚拟节点，解决数据倾斜
最小迁移增删节点只影响 ~1/N 的数据
