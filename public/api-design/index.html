3 URL 设计

资源命名规则

  • 使用名词而非动词：/users 而非 /getUsers
  • 复数形式：/users 而非 /user
  • 层级关系：/users/123/orders

常见模式

操作URL
获取用户列表GET /users
创建用户POST /users
获取指定用户GET /users/:id
更新用户PUT /users/:id
删除用户DELETE /users/:id
用户的订单GET /users/:id/orders

过滤与分页

?page=1&limit=20
?status=active
?sort=name&order=asc

常见错误

❌ 错误✅ 正确
GET /getUsersGET /users
/user/123/users/123
嵌套过深 /a/1/b/2/c/3扁平化或查询参数
所有错误返回 200正确使用 4xx/5xx

4 状态码

成功

状态码说明
200OK，成功
201Created，创建成功
204No Content，成功但无返回

客户端错误

状态码说明
400Bad Request，参数错误
401Unauthorized，未认证
403Forbidden，无权限
404Not Found，资源不存在
422Unprocessable Entity，验证错误

服务端错误

状态码说明
500Internal Server Error
503Service Unavailable

5 响应格式

成功响应

✅ JSON 格式

{ "code": 200, "message": "success", "data": {...}, "pagination": { "page": 1, "limit": 20, "total": 100 } }

错误响应

❌ 错误格式

{ "code": 400, "message": "参数错误", "errors": [{ "field": "email", "message": "邮箱格式不正确" }] }
